# ADR-002 — 하네스·스킬의 제작자와 검사자를 분리한다

- 상태: 승인됨 (사람 묵시 승인 후 실증, 2026-07-11)
- 일시: 2026-07-11
- 제안: 검수자 세션(Claude)
- 근거 문서: 계획서 §0.4 · FAIL-2026-012

## 1. 상황
검수자(Claude)가 하네스를 직접 만들다 같은 실패계열을 3회 반복했다. 특히 `gate-audit`은 `[loop]` 커밋 접두사를 "자동 주체"로 오판한 위에 세워져 **무인 결재 위반 22건을 날조**했고 철회됐다. 오늘 `claim-check`(역시 검수자 제작)의 정규식 결함이 **파이프라인 전체를 정지**시켰다.

**만든 사람이 검증하면 같은 착각이 그대로 코드에 새겨진다.**

## 2. 선택지
(A) 검수자가 계속 제작하고 조율자가 검사 — 지금까지의 방식. 실패 3회.
(B) **코덱스가 제작(`server/Harness/`·`skills/` 배타 소유), 조율자가 검사, 검수자는 지시서만 쓴다.**
(C) 사람이 하네스를 검토한다 — 병목.

## 3. 선택
**(B).** 코덱스는 FAIL 위키를 가진 주체다 — **자기가 등록한 실패를 자기가 하네스로 굳힌다.**

## 4. 판단 기준
독립 관찰자 원칙. 제작자의 인지 편향이 검증 도구에 상속되는 것을 차단.

## 5. 결과
- `server/Harness/`·`skills/`는 코덱스 배타 쓰기 영역. `server/Cli/CliRouter.cs`는 코덱스 금지(HOOK-01의 `HarnessRegistry`가 1회만 훅).
- **실증**: 코덱스가 H-00·H-0·H-01·H-6(자기 하네스 버그 수정)을 연속 산출했고, 조율자가 검수·커밋했다. 사람도 검수자도 개입하지 않았다.

## 6. 되돌림 조건
코덱스가 15분 루틴에서 큐를 픽업하지 못하면(한도·인증·프롬프트 미도착) 회차가 빈다. 3회 연속 공백이면 원인을 배제한 뒤(QUOTA-POLICY §1) 재검토한다.
