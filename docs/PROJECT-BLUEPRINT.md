# PROJECT-BLUEPRINT — 루프 시스템 진행 청사진

> 이 문서는 프로젝트 전체의 지도다. 개별 프로젝트의 목표는 각 blueprint.json이,
> 원칙의 이유는 DECISIONS.md가, 작업 방법은 skills/가 담당한다.
> 여기는 "어디서 왔고, 어디에 있고, 어디로 가는가"만 적는다.

## 한 줄 정의

**목표(청사진)를 선언하면, 로컬 AI들이 측정→제안→상호 검토를 자율 반복해
기준을 통과한 답안만 사람 결재에 올리는 로컬 우선 워크 시스템.**
단, 기준 변경과 최종 결재는 구조적으로 사람만 가능하다.

핵심 논지: **강한 게이트가 있으면 싼 실행자로 충분하다.**
통제 장치를 전부 먼저 짓고, 그 안에서 자율을 켰다 — 개발 순서 자체가 증거다.

## 관통 원리 3개

1. **판단을 측정으로** — 체크리스트(예/아니오), 픽스처, 이력서. 검토자에 대한 판단까지 측정화한다.
2. **비싼 것은 조건부로** — 매 회차는 싸게(규칙·로컬), 상위 호출·사람은 승격 조건에서만. 성능 지표는 회차당 고정 비용.
3. **의도를 산출물에** — 계약(JSON), 계보(서명·revisionOf), 기록(로그·incident·DECISIONS). 머릿속 판단은 루프가 재사용할 수 없다.

운영 철학 원문: *"꼭 내가 정해줘야 하는 것만 우선순위 붙여서 주고, 나머지는 알아서."*
이 시스템은 그 프롬프트의 인프라화다.

## 권한과 이양 (구 "사람 전용 불변")

어떤 권한도 원리적으로 사람 전용은 아니다 — 이양의 조건이 있을 뿐이다:
해당 권한에 대한 실적(이력서·그림자 검증) + 백스톱(롤백·재측정·사후 감사) + 예산 캡.

현재 사람 전용 (이양 조건 미충족):
- 기준·울타리 변경 (blueprint, definition, 레버 범위, 검사 완화)
- outbox 코드 diff 반입 / 스킬·규범 승급 / medium 이상 결재 / API 종량제 개방

고정점 (이양 불가 — 재귀의 바닥):
**권한을 이양한다는 결정 자체는 항상 사람이 한다.**
시스템이 자기 권한을 스스로 늘릴 수는 없다.

## 진행 지도

### 0기 — 목업과 운영 모델 ✓ 완료

관제 UI(3열+타임라인), 멀티 프로젝트(폴더=프로젝트), 상태 체계
(loopState/blocked 3종/lifecycle), 게이트·가드레일·체크포인트,
review-report 계약(사람/AI 동일), 위험도 재산정, diff 뷰, 이벤트 로그
(기계 키+사람 템플릿), 한/영·다크모드. Vanilla JS, 외부 의존 0.
**의미: 지능 0% 상태에서 통제 장치 100%를 먼저 완성.**

### 1기 — 실데이터 생산 라인 ✓ 완료 (전 항목 실측 검증)

- **C# 서버**: 영속화, 원자 쓰기, 복원 지점, 파손 자동 복구, 결재 데이터 자동 커밋
- **측정 루프**: dev-pack(저장소 자가 검사 12지표) / ruined-lab(시드 고정 전투 시뮬 500회)
- **탐색기**: 레버 울타리 안 그리디+shaping+스텝 확대, dry-run 채점(예측/사실 분리)
- **로컬 멀티 에이전트**: 생성 qwen3:8b ↔ 검토 qwen3:14b(신원 분리),
  체크리스트 1층(예/아니오만, uncertain=위로), needs_changes→재생성 1회
- **안전**: 악화 감지→suspend→롤백 proposal(자동 롤백 없음), 게이트 3형제
  (지시/재지시/산출물), 장애 파이프라인(incident→반복→승급)
- **에이전트 관례**: CLAUDE.md/AGENTS.md, 스킬 도메인 분리+경로 트리거 라우팅,
  커밋 전 measure 게이트
- **dispatch/outbox**: 사본 격리 실행, git 무접촉, diff 반입은 사람 결재
- **사다리 실측**: ollama 시도→게이트 탈락→claude-code 승격 (executor.escalated 실데이터)

**실증된 대표 기록**: 첫 닫힌 루프(14b 지적→8b 반영→승인) / 시스템이 실결함을
스스로 발견(smallTouchTargets) / "레버 범위 내 해 없음" 정직 보고→사람이 울타리 확장 /
검토자가 재생성 오염을 적발 / 코덱스 할당량 소진→강등 실증.

### 현재 위치 — E2E 최종장 + 무인화 전환

**진행 중**: 레버 확장(enemies.hp/count, 사람 승인 완료) 후 재탐색 —
completionRate 0%가 밴드 [45,60]에 도달하는지의 최종 명제 검증.
**사람 대기**: outbox 리팩토링 diff 반입 결재.
**투입 순서 확정**: #0 레버 후속 → #9 AI 1급 사용자(AGENT-GUIDE, contributions,
기본 경로를 dispatch로) → #10 AI 결재자(low 한정, claude-code, pacing 연동 —
무인 회차의 시작) → #7 회고 큐(유휴 훈련 1세대) → #8 규범 스토어.

E2E 성공의 정의(어느 쪽이든 유효): 밴드 도달 = "무료 로컬만으로 목표 수렴" 증명 /
no_solution = "한계를 정직히 보고하고 기준을 사람에게 넘기는 통제" 증명.
판정 명제 4: 수렴 여부 / 총비용 $0·구독 회계 / 사람 개입=결재·ack만 / 통제 장치 발동 기록.

### 2기 — 해동 조건부 (실적 데이터가 열쇠)

동결 이유는 전부 "재료 부족"이었고, 재료가 쌓이는 순서대로 깨어난다.

- **회고 큐** (#7로 해동 확정) — 자기 기록 요약+개선 초안, 환각 방어(근거 시각 인용 강제)
- **이력서** — 실행자×스킬×작업 유형별 실적(일치율/오탈/비용, minSamples).
  배정·승격·강등·autoApprove 확대를 예/아니오 조회로. 재료: 게이트 기록 JSON,
  자동 결재 뒤집기, 승격 이벤트 — 이미 축적 시작됨
- **그림자 모드** — 신규 검토자·정책은 판정권 없이 병행, 실적으로 승급
- **스킬 버전 토너먼트·유휴 훈련 전체** — 픽스처 채점, paused/aligned에서만, 시간 예산 캡
- **하네스 자동화 완성** — reasonCode 임계→차단형은 구조 졸업, 판단형은 대안 토너먼트
- **결정 게이트 브리핑** — 사람 대기 시간에 대안 후보+dry-run 비교표

### 3기 — 확장과 공개 (필요가 생길 때)

- **게임 본체**: Unity 세로 슬라이스(JSON 임포터+지표 수집) — 시뮬과 측정 계약 공유,
  예측 vs 실플레이 괴리로 시뮬 보정
- **원격 완성**: Tailscale 열람→결재(구현됨), 지시 제출(dispatch UI 입력창)
- **시연**: demo-pack(리셋 가능 시연 전용), 라이브 대본(위반 주입→루프→$0),
  GitHub Pages mock, README 궤적 표
- **규범 내보내기**: 스토어(#8) — 성적표 딸린 팩만, 검증 전은 회색
- **새 도메인 팩**: 문서 팩, 에셋 팩(이미지·음악 — 형식 검사 0층+비전 체크리스트 1층,
  코어 무변경이 범용성의 최종 증명)
- **신입 에이전트 온보딩 테스트** — AGENT-GUIDE만으로 새 모델이 사용 가능한지
- **메타**: 이 시스템의 개발 자체가 이 시스템 위에서 (dispatch가 기본 경로화되며 진행 중)

### 언젠가 (기록만)

MCP 서버화 / 패턴 전파(스킬 간 교차 학습) / 회사 스케일 논증(문서 한 단락 유지) /
자동 트리거 회고 / restore·outbox 수명 정리 자동화.

## 미결 결정 (열려 있는 것)

- autoApprove의 maxRisk를 medium까지 올릴 조건 (이력서 데이터 후)
- 재생성 상한(현 1회)의 확대 시점 (통과율 실측 후)
- 문턱형 지표의 동반 지표 규칙을 도메인 팩 필수로 승급할지 (#6 결과 후)
- 검토 체크리스트의 낡은 항목 제거 (제안 목록 존재, 사람 결재 대기)

## 이 문서의 관리

큰 단계가 닫히거나 미결이 해소될 때 갱신한다. 갱신도 변경이므로
커밋 메시지에 사유를 남긴다. 상세 이력은 DECISIONS.md와
docs/verification/이 담당하며 이 문서는 항상 "현재의 지도"로 유지한다.
