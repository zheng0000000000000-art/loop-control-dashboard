# ADR-001 — 운영 등급을 `Required Before Multi-model Parallel Work`로 승격한다

- 상태: **사람 승인 대기**
- 일시: 2026-07-11
- 제안: 검수자 세션(Claude)
- 근거 문서: `docs/plan/AI-RUNTIME-REFACTOR-MICRO-DIRECTIVES-v9.md` §0-A.6 · `docs/plan/ALIGNMENT-v9.md`

## 1. 상황

계획서는 운영 규칙을 4등급(`Required Now` / `Required Before Automation` / `Required Before Multi-model Parallel Work` / `Optional`)으로 나누고, 처음부터 전부 적용하지 말라고 한다. 승격 조건 8개 중 **2개 이상**이면 한 단계 올린다.

실측 결과 **8개 중 7개**가 이미 충족돼 있다(ALIGNMENT-v9 §1). 요약:

- 실행자 4명이 병렬로 작동 중(sonnet·코덱스·조율자·검수자)
- 동일 파일 충돌 2회(`Tier2Approver.cs`·`Tier2ApproverTestCli.cs`)
- 인수인계 실패 2회(FAIL-2026-013 지시 미도착 · 조율자의 stale 사본 열람)
- stale 문서로 인한 잘못된 판단(STATUS가 삭제된 파일을 "준비됨"이라 기록)
- 자동 Scheduler 실행 중(5분·15분)
- push로 canonical repository에 실제 영향 발생

**즉 우리는 이미 최고 등급 상태로 운영되고 있으면서, 그 등급이 요구하는 안전장치를 하나도 갖추지 않았다.** 오늘 발생한 사고 6건이 그 공백과 일대일로 대응한다.

## 2. 선택지

**(A) 등급을 공식 승격하고, 그 등급의 필수 항목을 Phase 0로 세운다.**
- 장점: 사고의 원인을 구조적으로 제거. 계획서의 예방책을 그대로 사용(재발명 비용 0).
- 단점: 착수 비용. 당분간 기능 개발이 멈춘다.

**(B) 등급을 낮춘다 — 병렬·자동화를 끈다(실행자 1명, 스케줄 중단).**
- 장점: 안전장치 없이도 안전해진다. 계획서가 말하는 "초기에는 최소 통제".
- 단점: 지금 얻고 있는 자율 루프(코덱스 QA·조율자 검수)를 잃는다.

**(C) 현행 유지 — 사고가 나면 그때그때 대응한다.**
- 장점: 없음(오늘 하루가 반례다).
- 단점: 같은 계열 사고가 반복되고, 주체 규명이 불가능하며, 결국 사람이 매번 개입해야 한다.

## 3. 선택

**(A).** 단, **필수 항목을 전부 만들지 않는다.** 계획서의 Phase 예산(신규 하네스 2·스킬 2 상한)을 지키고, 이미 자생한 동등물(`server/Harness/`, `skills/common/`, allowlist+`scope-check`)은 **매핑으로 흡수**한다. 진짜 공백 6개만 만든다(ALIGNMENT-v9 §3·§4).

## 4. 판단 기준

1. **실체 우선** — 문서가 아니라 실측(프로세스·exit code·git)으로 등급을 판정했다.
2. **재발명 금지** — 이름이 달라도 기능이 같으면 동등물로 선언하고 옮기지 않는다(옮기면 CliRouter 배선·코덱스 소유권이 깨진다).
3. **예산 준수** — 하루에 하네스 6개를 만든 오늘 같은 과잉을 반복하지 않는다.
4. **순서** — 상태 원본 신뢰(handoff-integrity) → 파생 문서 생성(Projection) → 입력 신선도(Context Pack) → 동시 쓰기 차단(claim). 사고의 인과 사슬 순서다.

## 5. 결과

- Phase 0(P0-01~07)을 발행한다. 상세: `docs/plan/ALIGNMENT-v9.md` §4.
- 신규 하네스는 `handoff-integrity`·`context-pack-integrity` **둘로 제한**한다.
- `HS-GATE-P00` PASS 전에는 Phase 1(기능 개발)로 넘어가지 않는다.

## 6. 되돌림 조건

- Phase 0 착수 비용이 실익을 넘어선다고 판단되면 **(B)로 전환**한다 — 스케줄 태스크를 끄고 실행자를 1명으로 줄이면 등급이 자동으로 내려간다.
- 하향 조정은 "안전 요구가 줄었다는 근거"와 함께 **사람이 명시 승인**한다(§0-A.6).
