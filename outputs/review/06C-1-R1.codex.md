### 지표는 통과하지만 목적을 배신하는 경로
- 완료 기준 #6을 “이미 적용된 T-TEST-001 envelope의 self hash만 위조”로 시험하면 통과한다. 하지만 새 전이 envelope의 `transitionContractSha256`만 위조하면 self-hash 검사가 실행되지 않는다. 목적상 “envelope 자기신고 불신”이 미완성이다.

- 완료 기준 #2를 `state-transition --human-decision ...` 같은 bare legacy 형태만 시험하면 exit 2가 나온다. 하지만 `prepare/apply` 뒤에 붙은 `--human-decision`/`--root`는 usage 실패가 아니라 silent ignore다. CLI 표면에서 옵션이 사라졌다는 목적을 배신한다.

- 완료 기준 #13은 `outputs/launch/`만 찌르면 통과한다. 하지만 broad historical prefix 아래 새 운영 문서가 생기면 legacy 호출을 숨길 수 있다. “활성 경로의 옛 호출을 잡는다”가 아니라 “특정 샘플만 잡는다”가 된다.

### 테스트가 못 잡는 것
- 새 전이 self-hash 위조:
  1. 정상 `state-transition prepare --transition-id T-NEW --request req.json`.
  2. 생성된 `T-NEW.envelope.json`에서 `transitionContractSha256`만 `"aaaa"`로 변경.
  3. `state-transition apply --envelope T-NEW.envelope.json`.
  기대 반증: `exit 1 envelope-contract-mismatch`.

- 삭제 옵션 silent accept:
  - `state-transition prepare --transition-id T-OPTS --request req.json --human-decision approved.json`
  - `state-transition apply --envelope T-OPTS.envelope.json --root C:\somewhere`
  기대 반증: 둘 다 `exit 2 + usage`.

- prefix allowlist 우회:
  - `docs/wiki/live-runbook.md` 내용: `state-transition --transition-id T --expected-workstate-sha256 S --request r.json`
  - `state-transition-callsite-check`
  기대 반증: `exit 1`, `legacyCallsiteCount >= 1`.
