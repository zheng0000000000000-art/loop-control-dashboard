# 코덱스 독립 검수 — 05H-R2 (pending 면제)

- 검수자: codex (read-only 샌드박스). 대상: HandoffIntegrityChecker/Cli + pending fixture 5종.

> **검수자 대조 결과 — 코덱스 주장 2건 중 1건 사실, 1건 오류.**
>
> **사실**: `--self-test`가 실패 *코드*를 단언하지 않는다. 검수자가 실증했다 —
> `pending-duplicate/workstate.json`을 malformed로 만들면 기대(`duplicate-in-state`)와 다른 이유(`workstate-malformed`)로
> 실패하는데도 `--self-test`가 **exit 0 / verdict PASS**를 준다.
>
> **오류**: "fixture JSON이 malformed다" — **실제 fixture 5종은 전부 valid JSON이다.**
> **원인은 검수자다.** `run-codex-review` 입력 패킷을 PowerShell `Out-File -Encoding utf8`로 만들어 BOM이 붙고 한글이 깨졌다.
> 코덱스가 인용한 `state??` 의 `??`가 그 증거다. **독립 검수자에게 준 입력이 오염되면 판정도 오염된다.**

---

### 판정
FAIL — pending 면제 조건 자체는 닫혔지만, `--self-test`가 기대 실패의 원인/코드를 단언하지 않아 malformed/missing fixture도 PASS 도장으로 처리할 수 있다.

### 남은 우회로
- checker 면제 조건: 남은 우회 입력은 확인하지 못했다. `allLogIdSet`을 만들고 (`HandoffIntegrityChecker.cs:188`) pending 조건에 `!allLogIdSet.Contains(id)`를 추가했다 (`HandoffIntegrityChecker.cs:254-257`). 따라서 같은 exact `transitionId`의 실패 로그가 있으면 면제되지 않고 `state-transition-not-logged`로 떨어진다 (`HandoffIntegrityChecker.cs:262-263`).
- self-test 우회: 기대 FAIL 케이스의 fixture 파일을 삭제하거나 malformed로 만들어도 `actualPass=false`, `actualExemption=false`만 맞으면 PASS 처리된다 (`HandoffIntegrityCli.cs:361-384`). 예: `pending-failed-log/workstate.json` 파일 없음 또는 invalid JSON.

### fixture가 못 잡는 것
- 기대 실패의 “원인”을 못 잡는다. `pending-duplicate`가 `duplicate-in-state`가 아니라 `workstate-malformed`여도 self-test는 통과 가능.
- `pending-mismatch`가 실제로 state-to-log 경로에서 `state-transition-not-logged`를 내는지 못 잡는다. 현재 제공 내용 자체가 malformed다 (`PENDING-FIXTURES.txt:42`).
- 추가로 빠진 pending case:
```json
// workstate.json
{
  "appliedTransitions": [
    { "id": "PENDING-FAILED", "appliedAt": "2026-01-01T00:00:00Z" }
  ]
}
// applier-log.jsonl
{"transitionId":"PENDING-FAILED","result":"error-but-zero","exitCode":0,"at":"2026-01-01T00:00:01Z"}
```
기대: pending 면제 미적용, `state-transition-not-logged`. 현재 코드는 잡지만 fixture 5종에는 “result != ok, exitCode == 0” 조합이 없다.

### 확인 못 한 것
- 실제 `dotnet build`, `handoff-integrity --self-test` 실행 결과는 확인 못 했다. 제공 workspace에는 `*.csproj`와 실제 `docs/qa/fixtures/...` 트리가 없고 read-only 검수 산출물만 있다.
- `--pending-transition` CLI 금지는 소스상 유지됨: `HandoffIntegrityCli.cs:27-30`.
- `--self-test`에 canonical 데이터를 먹이는 통로는 소스상 없음: `--self-test` 감지 후 `ParsePaths` 전에 `RunSelfTest(root)`로 바로 간다 (`HandoffIntegrityCli.cs:33-35`), fixture 경로는 하드코딩 (`HandoffIntegrityCli.cs:356-359`).
