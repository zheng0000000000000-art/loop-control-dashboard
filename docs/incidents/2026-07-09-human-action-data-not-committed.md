# 사람 액션 데이터 커밋 누락

## 증상

사람 결재가 만든 `workflow-state.json`, `game-data.json`, `run-log.json`, `review-report.json` 변경이 로컬에 남고 push에서 빠질 수 있었다.

## 원인

사람 액션은 서버가 파일에 기록했지만, 운영 데이터 커밋은 에이전트나 사람이 별도 절차로 수행해야 했다. 결재 직후 커밋이라는 시스템 경계가 없었다.

## 해소

- 서버가 사람 액션 저장 직후 `dashboard/data/**` 변경만 자동 git 커밋한다.
- 코드 파일은 자동 커밋 대상에서 제외한다.
- `Git:AutoPush` 기본값은 `false`로 두고, 필요한 환경에서만 자동 push를 켠다.
- git 실패는 콘솔 로그로 남기고 결재 처리는 막지 않는다.

## 재발 방지

운영 데이터의 커밋 책임은 서버가 맡고, 코드 변경 커밋은 에이전트나 사람이 맡는다.
