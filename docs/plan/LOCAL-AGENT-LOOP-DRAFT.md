# 로컬 에이전트 루프 — 설계 초안 (검수자, 2026-07-12)

> **상태: 초안. 사람 결재 전.** 이 문서는 **제안이지 결정이 아니다.** ADR로 승격하기 전에 사람이 방향을 고른다.
> 근거: `ADR-011`(Phase 0 완료 = 로컬 AI가 프로그램의 도움으로 DI를 완수) · `INTENT-DIGEST §4-B`

---

## 1. 문제를 정확히 말하면

**ollama는 지금 에이전트가 아니라 생성기다.** `OllamaExecutor.Generate()`는 note 한 줄을 뱉는다. **파일을 읽고·고치고·하네스를 돌리는 한 바퀴가 없다.**

그런데 **한 바퀴가 없으면 `ADR-011`의 시뮬레이션 자체가 불가능하다.** 이게 Phase 0의 마지막 벽이다.

### 오늘 잰 숫자 (이게 설계를 결정한다)

| 사실 | 값 |
| --- | ---: |
| 상위 모델 실행자 1회 | **49,281 토큰** (그중 실제 작업 **193**) |
| ollama 제안 생성 1회 | **446 토큰** |
| 로컬 RESUME 시험 | **580 토큰** — sonnet과 **동일한 답**, 82배 저렴 |
| 로컬 RULES 시험 | **3,800 토큰** — **거부는 맞혔으나 근거를 날조** |

**두 가지가 나온다:**

1. **짐이 문제지 머리가 문제가 아니다.** 좁은 과제에서 로컬은 상위 모델과 동등했다.
2. **로컬 모델의 설명은 증거가 될 수 없다.** 존재하지 않는 "규칙 1.1"을 지어냈다.

---

## 2. ★ 설계 원칙 — **범용 에이전트를 만들지 않는다**

보통의 ReAct 에이전트는 **모델이 "다음에 어떤 도구를 부를지" 결정**한다. **우리는 그렇게 하지 않는다.**

> **프로그램이 루프를 돌리고, 모델은 정해진 자리에서 판단만 한다.**
> (`INTENT-DIGEST`: *"LLM은 적게 기억하고 적게 생성한다. 프로그램이 많이 기억하고 조립하고 검증한다. LLM은 판단만 한다."*)

**왜 이 방향인가 — 오늘 실측이 답한다:**

- 모델이 도구 호출 JSON을 만들게 하면 **`functionsWithoutComment`를 `functionsWithOutComment`로 쓰는 그 실패가 도구 이름·인자에서 그대로 재현된다.** 가장 쉬운 계약에서 이미 실패했다.
- 모델이 "무엇을 읽을지" 정하게 하면 **컨텍스트가 폭발한다.** 49k의 정체가 그것이다.
- **모델의 자유도를 줄일수록 약한 모델이 쓸 수 있다.** 로컬화는 모델을 키우는 게 아니라 **자리를 좁히는 일**이다.

---

## 3. 제안하는 구조 — **프로그램 주도, 모델 보조**

```
[프로그램]  Context Pack을 읽는다 (readOrder · requiredInputs · allowlist)
   ↓        필요한 파일만 정확히 로드한다 ← 모델이 고르지 않는다
[프로그램]  작업을 '좁은 판단 하나'로 쪼갠다
   ↓
[모델]      그 하나만 답한다 (예: "이 함수에 붙일 한국어 주석 1줄")
   ↓
[프로그램]  결과를 스키마로 검증한다 → 실패하면 사유와 함께 재시도(N회)
   ↓
[프로그램]  파일에 적용한다 (allowlist 밖이면 거부)
   ↓
[프로그램]  하네스를 돌린다 (gate manifest의 기대 exit와 대조)
   ↓        FAIL이면 그 결과를 모델에게 피드백으로 주고 다시
[프로그램]  gate.json · transport evidence · verification 문서를 **프로그램이 채운다**
```

**핵심: verification 문서의 「사용한 하네스와 exit code」를 모델이 적지 않는다.** 오늘 로컬 모델이 규칙 번호를 날조한 것이 그 이유다. **프로그램이 찍는다.**

### 3.1 모델이 답하는 "좁은 판단"의 예 (전부 스키마로 검증 가능)

| 판단 자리 | 입력 | 출력 계약 |
| --- | --- | --- |
| 주석 생성 | 함수 본문 1개 | 한국어 1줄 |
| 패치 생성 | 파일 1개 + 위반 1건 | **unified diff 1개** |
| 실패 분류 | 하네스 stdout | enum 1개 |
| 재시도 판단 | 이전 실패 사유 | `retry` \| `abort` \| `escalate` |

**출력이 스키마에 안 맞으면 프로그램이 거부하고 재시도한다.** 모델의 순종에 기대지 않는다 — **ADR-008의 정규화 + 계속 기록**을 그대로 적용한다.

## 4. 세 갈래 선택지 (사람이 고를 것)

### (A) 자체 루프 — `server/LocalAgentCli.cs` (권장)

- ollama `/api/chat` + **도구 없이**, 프로그램이 루프를 돈다.
- 장점: **완전 통제.** 컨텍스트·재시도·검증·증거를 전부 우리가 쥔다. 기존 하네스·Context Pack·gate manifest를 그대로 재사용.
- 단점: 우리가 짠다.
- **예산**: 신규 하네스 아님(실행기다). `e2e-usage` 확장으로 시뮬레이션 하네스를 붙인다.

### (B) ollama tool calling 사용

- qwen3는 `/api/chat`의 `tools`를 지원한다. 모델이 도구를 고른다.
- 장점: 표준적. 적게 짠다.
- **단점: 모델이 도구 이름·인자를 틀린다.** 오늘 metricId 대소문자 실패가 그 예고편이다. **자유도가 늘면 약한 모델이 더 못 한다.**
- **판단: 2단계로 미룬다.** (A)가 안 되는 게 증명된 뒤에.

### (C) MCP로 하네스를 노출

- 하네스·Context Pack을 **MCP 서버**로 감싸면 어떤 클라이언트든 붙는다(Claude Code · 로컬 러너 · 외부 도구).
- 장점: **상호운용.** 상위 모델과 로컬 모델이 **같은 도구 표면**을 쓴다 → 비교 실험이 정직해진다.
- 단점: 레이어가 하나 는다. **지금 필요한 건 아니다.**
- **판단: (A) 이후. 다만 (A)를 짤 때 "나중에 MCP로 감쌀 수 있게" 인터페이스를 CLI로 깨끗이 둔다.**

## 5. 모델 구성 제안 (실측 기반, 사람 승인 필요)

지금 있는 것: `qwen3:8b` · `qwen3:14b` · `llama3.1:8b`

| 역할 | 제안 | 근거 |
| --- | --- | --- |
| **패치·코드 생성** | **`qwen2.5-coder:7b` 또는 `14b` 추가** | 지금은 코드 특화 모델이 **없다.** 범용 모델로 diff를 만들게 하는 건 불리하다 |
| 리뷰·판정 | `qwen3:14b` (유지) | tier1 리뷰에서 안정적으로 작동 중(1,541/147) |
| 짧은 판단·분류 | `qwen3:8b` (유지) | 싸고 충분하다 |
| **검색·컨텍스트 축소** | **임베딩 모델 추가 검토**(`nomic-embed-text` 등) | **짐을 줄이는 것이 로컬화다.** "필요한 파일만 로드"를 **프로그램이** 하려면 검색이 필요하다 |

**단, 모델 추가는 "일단 넣어보자"가 아니라 실측으로 정한다.** 같은 과제를 두 모델에 돌려 **토큰·정확도·계약 준수율**을 비교한다. **숫자 없이 모델을 늘리지 않는다.**

## 6. 단계별 계획 (각 단계가 끝날 때마다 숫자를 남긴다)

**1단계 — 최소 루프 (SIM 가능해지는 지점)**
- `local-di-run` CLI: Context Pack 읽기 → 좁은 판단 1회 → 스키마 검증 → 적용 → 하네스 → gate.json
- **대상 DI를 하나 고른다**: 가장 좁고 기계적인 것. 예: **"함수에 한국어 주석 1줄 붙이기"**(`functionsWithoutComment` 위반 해소). **판단 자리가 하나뿐이고, 검증이 완전히 기계적이다.**
- **성공 기준**: 로컬 모델이 그 DI를 **끝까지** 완수하고 `di-completion-check`가 **PASS**한다.

**2단계 — 컨텍스트 예산 실측**
- 그 루프가 모델에 **실제로 보낸 토큰**을 잰다. `LAUNCH-BUDGET.json` 숫자를 **여기서 나온 실측으로** 정한다(사람 승인).

**3단계 — 적대적·재개 시험을 로컬로**
- RULES / RESUME를 **하네스로 굳힌다**(`e2e-usage` 확장). **회귀로 계속 돈다.**

**4단계 — 범위 확대**
- 판단 자리를 하나씩 늘린다. **늘릴 때마다 계약 준수율이 떨어지는 지점을 기록한다.** 그 곡선이 "로컬로 어디까지 가능한가"의 답이다.

---

## 7. 정직한 위험

- **1단계 DI(주석 붙이기)는 너무 쉬울 수 있다.** 통과해도 "로컬로 DI를 할 수 있다"의 증명은 **아주 좁다.** 그 좁음을 명시해야 한다. **지표를 목적으로 삼지 않는다**(ADR-005).
- **모델의 설명을 증거로 받지 않는 구조를 만들어도, 사람이 그 설명을 읽고 믿을 수 있다.** 리포트에 **"모델 설명은 증거가 아님"**을 못박아야 한다.
- **재시도 루프는 토큰을 먹는다.** N회 재시도가 상위 모델 1회보다 비싸지는 지점이 있다. **그 손익분기를 재야 한다.**

## 8. 사람이 정할 것

1. **(A)/(B)/(C) 중 어느 갈래인가** — 검수자 권고는 **(A) 자체 루프**, MCP는 인터페이스만 열어두고 나중에.
2. **모델을 추가할 것인가** — 코더 모델·임베딩 모델. **추가 전에 비교 실측을 할 것인가.**
3. **1단계 대상 DI** — "주석 붙이기"로 충분한가, 더 어려운 걸 골라야 하는가.
