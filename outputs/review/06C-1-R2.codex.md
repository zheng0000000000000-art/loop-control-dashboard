### 판정
FAIL — `prepare/apply`와 `--self-test`의 CLI 표면이 아직 완전히 닫히지 않아, 알 수 없는/삭제된 인자를 조용히 무시하는 경로가 남아 있다.

### 지표는 통과하지만 목적을 배신하는 경로
- 어떻게 통과시키는가: 완료 기준의 `--bogus-flag zzz`처럼 “값 있는 unknown option”만 시험한다.  
  무엇이 배신되는가: CLI 표면을 닫는 목적은 모든 낯선 인자를 호출자가 알게 하는 것인데, 값 없는 플래그와 위치 인자는 여전히 조용히 사라진다.  
  지금 구현에 실재하는가: 실재한다. `ParseFlagMap()`이 `--key value` 형태만 map에 넣고 나머지를 무시한다.

- 어떻게 통과시키는가: `state-transition --self-test`만 실행해 exit 0을 확인한다.  
  무엇이 배신되는가: self-test가 “인자를 받지 않는 고정 반증기”여야 하는데, 인자를 붙여도 성공한다. 호출자는 특정 root/id를 시험했다고 오해할 수 있다.  
  지금 구현에 실재하는가: 실재한다. `Run()`은 `args[2..]`를 검사하지 않는다.

### 테스트가 못 잡는 것
- 값 없는 unknown option:
  `state-transition prepare --transition-id T --request nonexistent.json --bogus-flag`  
  기대 반증: `unknown-option: --bogus-flag`, exit 2. 파일 존재 여부 오류가 먼저 나오면 실패.

- 위치 인자 쓰레기:
  `state-transition prepare --transition-id T --request <valid-req.json> zzz`  
  기대 반증: usage/unknown argument, exit 2. 현재 구조상 `zzz`는 버려진다.

- 값 없는 removed option:
  `state-transition apply --envelope <valid.envelope.json> --root`  
  기대 반증: `removed-option: --root`, exit 2. 현재 구조상 `--root`는 값이 없으면 버려진다.

- self-test extra args:
  `state-transition --self-test --root C:\copy --transition-id T`  
  기대 반증: self-test는 인자를 받지 않는다고 거절. 현재 구조상 temp fixture 테스트가 실행된다.
