## 고칠 것 여섯

### 1-1. legacy 단일-샷 경로 **삭제**

`--transition-id / --expected-workstate-sha256 / --request / --human-decision` 형태의 옛 경로 **전체 제거**.

- **`--human-decision`과 `ValidateHumanDecision`을 완전히 지운다.**
  **이 구멍으로 검수자가 실제로 자기 승인을 위조했다.** AI가 임의 JSON을 써서 phase 전이를 통과시켰다.
- legacy의 rollback 부재(`:560-612`)·가짜 멱등(`:695-715`)은 **경로와 함께 사라진다. 따로 고치지 마라 — 지우면 된다.**
- 남는 CLI 표면은 **`state-transition prepare` / `state-transition apply` 둘뿐.** 그 외 인자 조합은 **exit 2 + usage.**

### 1-2. contract hash를 **재계산**하라 (`:681-685`)

지금은 **envelope가 스스로 신고한 `transitionContractSha256` 문자열을 그대로 log와 비교한다.**
→ **성공 로그의 hash를 복사해 다른 request의 envelope에 넣으면 `idempotent exit 0`.**
**결함 4가 미수정이다. 결속하는 척한다.**

envelope의 **구성 필드로부터 직접 계산**하고 그 계산값을 log와 비교하라.
`envelope.transitionContractSha256`은 이제 **신뢰 입력이 아니라 대조 대상**이다 —
계산값과 다르면 `exit 1 envelope-contract-mismatch`.

### 1-3. test seam에서 **환경변수를 제거**하라 (`:357-361`)

주석은 *"production 노출 플래그 아님"*이라고 쓰여 있는데 **환경변수를 읽는다. 사실과 반대다.**
`#if DEBUG` 가드가 없어 **Release 바이너리에 그대로 들어간다.** 환경변수는 **자식 프로세스로 상속된다.**
**주석으로 계약을 만족시킬 수 없다.**

→ **in-process 테스트 훅**(`internal static Func<bool>?` 등)으로 바꿔라. 같은 프로세스의 자기 시험 코드만 설정 가능해야 한다.
**production 진입점(CLI 인자·환경변수·설정 파일) 어디서도 켤 수 없어야 한다.**

### 1-4. `--root`를 **제거**하라 — 사본 시험은 **cwd로 한다**

내 프롬프트가 명시적으로 금지했는데(`06C-1.prompt.txt:63`) 만들어졌다.
`canonicalMode = string.IsNullOrWhiteSpace(opts.Root)`(`:214`)는 **Root를 canonical 루트와 대조하지 않는다.**
→ **`state-transition apply --root <canonical repo>`** 로 부르면
**WORKSTATE는 쓰면서 projection을 건너뛴다 → `RUNTIME-INDEX.md`(L0)가 조용히 낡는다.**
CLAUDE.md가 *"손으로 쓴 문서보다 이걸 믿어라"*라고 한 그 파일이다.

**`--root`와 `canonicalMode`를 삭제하라.** root는 항상 `FindRepoRoot()`. **projection은 항상 실행한다.**

> **★ 사본에서 시험하는 법 — 전임자가 몰랐던 것 (이게 `--root`를 만든 진짜 이유일 것이다)**
>
> **PowerShell의 `Set-Location`은 .NET의 `CurrentDirectory`를 바꾸지 않는다.** 그래서 사본으로 "이동"해도
> `FindRepoRoot()`가 원본을 찾는다. **진짜 cwd를 바꿔라:**
>
> ```
> cmd /c "cd /d <copy> && dotnet run --project server -c Release -- state-transition prepare ..."
> ```
> 또는 `Start-Process -WorkingDirectory <copy>`.
>
> **cwd가 사본이면 `FindRepoRoot()`도 `ProjectionCli.cs:32`도 전부 사본을 본다. 우회할 필요가 없다.**
> **시험마다 사본을 새로 만들어라.** 앞 시험이 사본 상태를 바꾼다.

### 1-5. unknown `transitionKind`를 **fail-closed** 하라

`"EVIL"` 같은 미지값이 **high-risk가 아니라고 판단되어 NORMAL처럼 진행된다.** fail-open이다.
→ `transitionKind ∉ {NORMAL, PHASE_CHANGE, RECOVERY, REPLAY}` → **exit 1 `unknown-transition-kind`.**

### 1-6. `state-transition-callsite-check`의 **범위 결손**

`legacyCallsiteCount=0`이 나온 이유의 절반이다:
- `outputs/launch/**` **전체를 historical allowlist로 빼버렸다.** 거긴 **활성 범위**다 — 발사 프롬프트가 산다.
- `.md`(운영 prompts/docs)·`.github/workflows/**`·루트 JSON/YAML을 **스캔하지 않는다.**

원 지시서 §7 범위대로 되돌려라. **historical allowlist는 경로 접두사가 아니라 명시 파일 목록**이어야 한다
(`docs/handoff/sessions/**`·`docs/handoff/queue/**`·`outputs/*.out.log` 같은 **증거·과거 문서만**).

> **`legacyCallsiteCount`를 0으로 만들려고 범위를 줄이지 마라. 그게 이번 반려 사유의 핵심이다.**
