# ADR-012 — **모델을 빼고도 같은 결과가 나오면, 로컬 AI가 한 게 아니다** (무모델 대조군 의무)

- 상태: **사람 승인 대기**
- 일시: 2026-07-12
- 제안: 검수자 세션(Claude). **2차 외부 검수 보고서**가 문제를 짚었고, 검수자가 실체로 대조해 채택했다.
- 관련: `ADR-005`(지표 vs 목적) · `ADR-011`(Phase 0 완료 기준 = 로컬 AI가 프로그램의 도움으로 DI를 완수한다) · `docs/plan/LOCAL-DI-RUNNER-DRAFT-v3.md` §3

## 1. 상황

`ADR-011`은 Phase 0의 완료를 **"로컬 AI가 프로그램의 도움으로 DI를 완수한다"**로 재정의했다.
그런데 v3 설계에서 SIM-1은 **프로그램이 2~4개 후보를 만들고, 하네스가 정답을 판정하고, 모델은 `candidateId`를 고른다.**

**그러면 모델 없이 후보를 차례로 시험해도 DI가 끝난다.**

증명되는 것은 **"프로그램이 DI를 완수했고 모델이 옆에 있었다"**이지, `ADR-011`이 요구한 문장이 **아니다.**
이것은 `ADR-005`가 이름 붙인 병 — **지표는 만족했으나 목적은 미달** — 의 정확한 사례다. 하네스는 green이고, 주장은 거짓이다.

## 2. 결정

**모델의 기여를 주장하려면 같은 frozen task·같은 격리 worktree에서 무모델 대조군을 함께 돌린다.**

| 경로 | 선택 방식 |
| --- | --- |
| **BASELINE-0** | 첫 후보를 무조건 선택 |
| **BASELINE-1** | deterministic heuristic 또는 전 후보 순차 시험 |
| **MODEL** | 로컬 모델이 후보 선택 |

**모델의 기여는 아래 중 하나가 실측될 때만 인정한다:**

1. deterministic baseline이 **실패한** 과제를 성공
2. 정답 후보를 **더 적은 적용·하네스 실행 횟수**로 선택
3. 토큰·시간을 포함한 **전체 비용** 감소
4. 잘못된 후보로 인한 **새 위반** 감소

**하나도 보이지 않으면 결론은 "모델은 장식이었다"이고, 그것도 정직한 결과다. 숨기지 않는다**(ADR-005의 자진 신고 절에 쓴다).

## 3. 강제 방법 (프롬프트가 아니라 코드로)

- `run-evidence.json`에 **`armId`(`BASELINE-0`|`BASELINE-1`|`MODEL`)**를 필수 필드로 둔다.
- **`MODEL` arm만 있는 run으로 DI 완료를 주장하는 전이는 거부한다** — `state-transition`의 `--verdict` 결속(현재 미구현)에서 검사한다.
- 대조군 결과는 **같은 `runId` 묶음**으로 저장한다: `outputs/local-di/<runId>/arms/<armId>/`.

## 4. 되돌리는 법

이 ADR을 폐기하려면 **"모델 없이는 이 DI가 끝나지 않는다"를 프로그램이 증명**해야 한다(예: 후보 생성이 불가능한 DI 유형). 그 증명 없이 대조군을 빼는 것은 **기준 변경**이고 **사람 결재**다(`BASELINE-CHANGES.md`).

## 5. 한 줄

> **모델을 빼도 같은 결과가 나오면, 그건 로컬 AI가 한 게 아니다.**
