# FAIL-2026-018 — **검수자가 프록시로 네 번 오판할 뻔했다** (FAIL-2026-012의 재발)

- 상태: 진행 중 (규칙은 있는데 지켜지지 않는다 — 하네스화 필요)
- 최초 발생일: 2026-07-11 (FAIL-2026-012)
- 최근 발생일: 2026-07-14
- 관련 DI: 05H · 06C-1 · 06C-1-R1 검수
- 구성요소: verification-harness
- failureClass: known_failure (FAIL-2026-012 계열)
- 심각도: 높음 (**검수자가 오판하면 아무도 못 잡는다** — 검수자를 검수하는 자가 없다)

## 발생 상황

CLAUDE.md는 명시한다:

> **"원인을 지목할 때: 프록시로 단정하지 마라. 커밋 접두사·타임스탬프 상관·에러 문구·정규식 매치는 증거가 아니다."**

**FAIL-2026-012가 이미 이걸 기록했다. 그런데 2026-07-13~14 하루에 검수자가 네 번 더 걸렸다.**
**세 번은 스스로 잡았고, 한 번은 실제 사고를 냈다.**

## 관찰된 증상과 영향

| # | 프록시 | 프록시가 말한 것 | 실체 | 결과 |
| --- | --- | --- | --- | --- |
| 1 | **`where codex` 무결과** | "코덱스가 없다" | **설치돼 있었다**(MS Store 앱). 그 다음엔 **"헤드리스 진입점이 없다"** → 실은 **npm에 있었고 안 깔았을 뿐**이었다 | 잘못된 ADR 근거로 예외를 승인했다 |
| 2 | **파일이 안 생겼다** | "샌드박스가 쓰기를 막았다" | **모델이 시도조차 안 했다.** 말로만 따랐다 | **강제로 쓰기를 시켜보고서야** 커널 거부(`UnauthorizedAccessException`)를 확인 |
| 3 | **`git status` = `M`** | "실행자가 allowlist 밖 파일을 고쳤다" | **blob 해시가 HEAD와 바이트 단위로 동일.** mtime만 바뀐 **stat 캐시 artifact** | **범위 이탈로 반려할 뻔했다.** `git update-index --refresh`로 사라짐 |

| 6 | **`LastWriteTime`이 오늘** | "오늘 FAIL 사례가 기록됐다" | `git checkout` **복구 때문에 mtime이 오늘로 바뀐 것.** 실제로는 **2026-07-11 이후 하나도 없다** | git으로 재서 진실 확인 |
| 7 | **정규식 `Get-Content` 매치** | "`run-executor.ps1`에 인코딩 미지정 `Get-Content`가 있다" | **`Get-ContentText`**(사용자 정의 함수)를 오탐한 것 | 없는 결함을 보고할 뻔했다 |
| 8 | **프로세스 CPU 0** | "실행자가 멈췄다" | **40초 창으로 재니 0.34초 증가** + Anthropic API로 ESTABLISHED 연결 2개. **정상 API 왕복** | 짧은 창으로 봤으면 "죽었다"고 오판 |

## 발생 이유

- **직접 원인**: 검수자가 **관측 가능한 신호**를 **실체**로 착각한다.
- **근본 원인**: **프록시는 싸고 실체는 비싸다.** `git status`는 즉시 나오고, blob 해시는 두 번 더 쳐야 한다.
  **압박이 있을 때 싼 쪽으로 기운다.**
- **기여 원인**: **검수자를 검수하는 자가 없다.** 실행자의 자기보고는 검수자가 반증하지만,
  **검수자의 자기보고는 아무도 반증하지 않는다.**

## 검토한 해결 방법

1. 규칙을 더 강하게 쓴다 (× — **이미 CLAUDE.md에 있고 FAIL-2026-012에도 있다. 규칙으로는 안 된다**)
2. **프록시→실체 대응표를 스킬로 만든다** (○)
3. **검수자의 판정에도 독립 검수를 붙인다** (○ — 코덱스가 이미 검수자의 요구 결함을 지적했다)
4. **자주 쓰는 프록시를 하네스가 막는다** (○ — 예: `git status` 대신 blob 해시를 쓰는 헬퍼)

## 선택한 해결 방법

**2 + 3.** `skills/common/root-cause-diagnosis.md`에 **프록시→실체 대응표**를 추가한다.

| 프록시 (싸다) | **실체 (정본)** |
| --- | --- |
| `git status` 의 `M` | **blob 해시**: `git hash-object <f>` vs `git rev-parse HEAD:<f>` |
| `where <cmd>` 무결과 | **탐색 범위를 함께 적어라.** PATH·Store 앱·npm·설정 디렉터리를 각각 확인 |
| 파일이 안 생겼다 | **강제로 시도시켜서 거부되는지 보라.** "안 했다"와 "못 했다"는 다르다 |
| BOM이 있다 | **왕복 시험.** 고친 뒤에도 깨지면 다른 곳이 범인이다 |
| `New-Item -Force` 성공 | **사전에 `Test-Path`로 존재 여부를 기록하라.** `-Force`는 이미 있어도 성공한다 |
| `LastWriteTime` | **`git log --since`** (파일 시각은 복구·체크아웃으로 오염된다) |
| 정규식 매치 | **매치된 줄을 눈으로 읽어라.** 부분 문자열이 다른 심볼일 수 있다 |
| 프로세스 CPU 순간값 | **30초 이상 창으로 델타를 재라.** + 네트워크 연결·자식 프로세스 확인 |
| 하네스 exit 0 | **하네스를 먼저 반증하라** (FAIL-2026-015) |

**그리고 검수자 자신에게 적용한다**: **"내가 지금 프록시를 보고 있는가?"를 판정 전에 묻는다.**

## 판단 기준

**"관측할 수 있다"와 "그것이 사실이다"는 다르다.**
**부재를 주장할 때는 탐색 범위를 함께 적어라.**

## 검증 결과

- 사례 3(`git status`): blob 해시로 재서 **무죄 확인** → 잘못된 반려를 막았다
- 사례 5(`docs/wiki`): **막지 못했다. 실제로 42파일을 지웠다.** `git checkout`으로 복구
- 사례 8(CPU): 40초 창 + 네트워크 확인으로 **정상 판정** → 실행자를 죽이지 않았다

## 재발 방지

1. **반증 시험용 임시 파일은 저장소 밖(`$TEMP`)에 만든다.** 사례 5는 저장소 안에서 시험하다 났다.
2. **`-Recurse` 삭제 전에 그 디렉터리가 원래 있었는지 `Test-Path`로 먼저 확인하고 기록한다.**
3. **판정 전에 묻는다: "이건 프록시인가 실체인가?"**
4. **검수자의 판정에도 코덱스 독립 검수를 붙인다.** 요구의 결함까지 지적하라고 프롬프트에 명시하라.

## 후속 작업과 잔여 위험

- `skills/common/root-cause-diagnosis.md`에 프록시→실체 대응표 추가 (**미수행**)
- **잔여 위험**: **이 사례 자체가 검수자의 자기보고다.** 여덟 건은 **알아챈 것**이고,
  **못 알아챈 프록시 오판이 오늘 몇 건 더 있었는지 모른다.** 코덱스 독립 검수가 유일한 외부 눈이다.
