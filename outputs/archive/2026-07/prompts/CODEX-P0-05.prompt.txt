### 판정 규칙

| 조건 | 판정 |
| --- | --- |
| `requiredInputs`의 경로가 **없는 파일** | `missing` → **exit 1** |
| 파일은 있으나 **sha256 불일치** | `stale` → **exit 1** |
| 전부 일치 | `ok` → exit 0 |
| Context Pack 블록 자체가 없음 | `skipped` → **exit code에 영향 없음** |

`skipped`가 실패가 아닌 이유: 과거 지시서 때문에 게이트가 영구 잠기면 안 된다(FAIL-2026-010, CRLF 게이트 교착).

### 왜 이 하네스가 필요한가 (실측 근거)

`docs/STATUS.md`가 "ORCH-01 참조 .cs 준비됨"이라고 적었지만 그 파일은 커밋 `797e7bc`가 **이미 지운 상태**였다. 발사 직전에 **사람이 손으로** 발견했다 — 하네스 6개가 돌고 있었는데 아무도 못 잡았다. 참조 입력에 **경로만 있고 해시가 없었기 때문**이다.

### ★ 설계 제약 두 가지 (반드시 지켜라)

1. **스탬핑 기능을 넣지 마라. 검사만 해라.**
   P0-04의 `handoff-integrity`는 **스탬핑 주체(`projection`)와 검증 주체가 같아서**, `projection`을 재실행하면 **언제든 green을 제조할 수 있다.** 이번엔 **고치는 자와 검사하는 자를 분리한다**(ADR-002와 같은 원리). 해시 갱신은 지시서를 쓰는 사람/검수자가 한다.
2. **`requiredInputs`(읽기 참조)와 allowlist(쓰기 대상)가 겹치면 경고를 내라.**
   작업 중 바뀌는 파일에 해시를 걸면 **게이트가 자기 작업에 걸려 넘어진다.**

### 장애 주입 시험 (필수 — 통과를 믿기 전에 실패시킬 수 있음을 먼저 증명해라)

1. `requiredInputs`의 파일 하나를 **없는 경로로 바꾼 사본**으로 → `missing`, **exit 1** 확인
2. 참조 파일을 **한 줄 바꿔서** → `stale`, **exit 1** 확인
3. 원복 후 → **exit 0** 복귀 확인

**원본 지시서를 건드리지 말고 사본(`$env:TEMP`)에서 주입해라.** 검수자가 2026-07-12에 `git checkout`으로 주입을 원복하다가 **미커밋 작업물을 통째로 날렸다.**
