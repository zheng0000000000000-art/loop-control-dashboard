# ADR-010 — 지시 도착 증명을 **말(ACK)에서 바이트(hash)로** 옮긴다 (ADR-004 supersede)

- 상태: **사람 승인 대기**
- 일시: 2026-07-12
- 제안: 검수자 세션(Claude). **외부 검수 보고서**(`AI_Runtime_Project_Review_Report_2026-07-12`)가 문제를 정확히 짚었고, 검수자가 실측으로 확인했다.
- **이 ADR은 `ADR-004`를 대체한다(supersedes).** ADR-004는 **상태가 "제안"이었고 확정된 적이 없다.**

## 1. 상황 (실측)

**FAIL-2026-013**: 발사 프롬프트가 명령행 인자 경계에서 잘려 **실행자가 지시서를 받은 적도 없이** 저장소를 읽고 딴 일을 했다(안전 보류 항목까지 구현). 그래서 "지시가 도착했는가"를 검사하는 `launch-check` 하네스를 만들었다.

**그런데 그 하네스는 모델의 순종에 의존했다:**

```csharp
// server/Harness/LaunchCheckCli.cs (교체 전)
var expectedAck = $"ACK-{taskId}";
var ackFound = text.Contains(expectedAck, StringComparison.Ordinal);
return exists && ackFound && !quotaSignal ? 0 : 1;
```

**실측 5/5**: ORCH-01·ACTOR-01·FIX-04·FIX-05·FIX-06 **전부 ACK를 출력하지 않았다. 그런데 5건 다 지시서를 정확히 지켰다 — 지시는 도착했다.**

→ **정상 산출물을 오탐으로 죽이는 게이트**였고, 그래서 **어떤 발사 경로에도 배선되지 않았다**(Registry 등록만, 자동 호출 0건). **비활성·비권위 게이트.**

**`ADR-004`가 고른 대안(파일 기반 ACK)도 같은 병이다** — 실행자가 파일을 써야 하므로 **여전히 모델의 순종에 의존한다.**

**이 프로젝트의 실증**: *"말은 안 지켜지고, 검사만 지켜진다."*

## 2. 선택지

**(A) ACK를 stdout에서 파일로 옮긴다** — `ADR-004`의 선택. **모델 의존이 그대로다.** 실행자가 파일을 안 쓰면 끝.

**(B) 전송 무결성을 CLI 경계에서 증명한다 (선택)**
프롬프트를 **stdin**으로 보내고, Claude CLI의 **`--replay-user-messages`**가 되돌려준 user 메시지의 sha256을 대조한다.
**모델은 관여하지 않는다.** CLI가 되돌려주는 것이다.

**(C) 아무것도 안 한다** — FAIL-2026-013이 재발한다. 지금도 프롬프트는 명령행 인자로 간다.

## 3. 선택: **(B)**

```
claude -p --verbose --input-format stream-json --output-format stream-json --replay-user-messages
```

**검수자 실측(2026-07-12)**: 설치된 CLI가 지원한다. stdin으로 보낸 본문이 그대로 되돌아왔고 sha256이 일치했다.

```
source   = 58A55C1CBCD3620F19331DDF...
received = 58A55C1CBCD3620F19331DDF...
→ 일치
```

## 4. ★ 이 결정의 권위 범위 — **넘겨짚지 마라**

**replay hash가 증명하는 것: "내가 보낸 바이트 = Claude CLI가 user message로 받은 바이트."**

**증명하지 않는 것:**
- 모델이 그 내용을 **읽었는가**
- 모델이 그 내용을 **이해했는가**
- 모델이 그 지시를 **따를 것인가**

→ 판정 문자열에 **`MODEL_RECEIVED`·`MODEL_UNDERSTOOD`·`ACK_VALID` 같은 표현을 쓰지 않는다.** 판정은 **`TRANSPORT_VALID` / `TRANSPORT_INVALID`** 둘뿐이다.

**이 절이 없으면 다음 사람이 "receipt 통과 = 지시 이행"으로 오해한다.** 그건 우리가 ACK로 이미 한 번 저지른 착각이다.

## 5. 판단 기준

1. **말이 아니라 바이트로 판정한다.** 모델의 협조가 필요 없는 검사만이 게이트가 될 자격이 있다.
2. **부작용으로 명령행 문제가 함께 사라진다.** stdin 전달이므로 **인자 길이·quote 경계·구조 손실이 원천 제거**된다. FAIL-2026-013을 **이번에는 진짜로 구조적으로 제거한다.**
   (※ 이전 기록에서 "파일 전달로 구조적으로 제거했다"고 적은 것은 **과장이었다.** 파일은 원본일 뿐, 전달은 여전히 명령행이었다. 정정한다.)
3. **fail-closed.** replay 누락·중복·파싱 실패·hash 불일치·옵션 미지원은 **전부 실패.** 실패한 실행의 산출물·보고·sentinel은 **유효한 산출물로 인정하지 않는다.**
4. **조용한 폴백 금지.** 명령행 방식으로 되돌아가는 경로가 남아 있으면 그 자체가 실패다.

## 8. 되돌림 조건

- CLI가 `--replay-user-messages`를 제거하거나 동작을 바꾸면 → **fail-closed로 전 발사가 막힌다.** 그때는 **사람이 운영 등급을 내리고**(자동화·병렬 중단) one-shot 수동 경로만 허용한다. **결과는 `PASS`가 아니라 `OVERRIDDEN`이다.**
- **override는 PASS 우회가 아니다.** 정상 승격은 **하네스를 고치고 다시 PASS를 받은 뒤에만** 가능하다.

- v9 Context Budget: **"Budget 초과 자체는 실패가 아니다."** 초과했는데 **분할 결정도 예외 사유도 없이 통과하는 것**이 실패다. 단순 상한 FAIL은 v9보다 강한 새 정책이므로 **사람 결재 대상**이다.
