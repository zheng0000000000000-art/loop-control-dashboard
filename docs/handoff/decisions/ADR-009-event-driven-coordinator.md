# ADR-009 — 조율자를 **시간 기반 → 사건 기반**으로 바꾼다 (할당량은 유한한 자원이다)

- 상태: **승인됨 (사람 choi, 2026-07-12 04:0x)** — 사람이 "실행자가 작업한 후에 확인하게 할 수도 있어? 시간이 아니라?"로 제기, 검수자가 구현.
- 적용: 예약 작업 `recursion1-result-check` 프롬프트에 **선(先) 게이트** 신설 + 발사 래퍼 `outputs/launch/run-executor.ps1` 신규.

## 1. 상황 (실측)

- 조율자는 **5분마다 무조건 LLM 세션을 하나 태웠다.** 할 일이 있든 없든.
- 실행자(sonnet)는 한 작업에 **15~25분**이 걸린다. 그동안 조율자는 **할 일이 없는데 4~5번을 태웠다.**
- **코덱스가 2026-07-11 23:37에 할당량으로 멈췄다**(하루 51세션). 사람이 "할당량이 다 되면 멈춘다"고 확인했다(ADR: `QUOTA-POLICY.md`).
- **할당량은 이 시스템의 진짜 병목이다.** 그런데 우리는 그것을 **아무 일도 없는 확인에** 쓰고 있었다.

## 2. 선택지

**(A) 주기를 늘린다**(5분 → 15분).
- 장점: 간단.
- 단점: **반응이 느려진다.** 실행자가 3분 만에 끝나도 12분을 기다린다. 낭비는 줄지만 **여전히 시간 기반**이다.

**(B) 사건 기반 (선택)** — 실행자 종료를 **신호로** 만들고, 신호가 없으면 조율자가 **즉시 종료**한다.
- 장점: **할 일이 있을 때만 태운다.** 반응은 오히려 빨라진다(종료 즉시 다음 회차가 잡는다).
- 단점: 발사 경로에 래퍼가 하나 늘고, 신호 파일의 수명 관리가 필요하다.

**(C) 조율자를 없애고 검수자가 다 한다.**
- 단점: **제작자와 검수자가 같아진다**(ADR-002 위반). 조율자는 독립 관찰자로 남아야 한다.

## 3. 선택: **(B)**

### 구현

**발사 래퍼** `outputs/launch/run-executor.ps1`:
1. 프롬프트를 **파일**(`<TaskId>.prompt.txt`)에서 읽는다 → **인자 경계에서 잘리는 사고(FAIL-2026-013)를 구조적으로 제거**한다. 지금까지는 "한 줄로 쓰라"는 **규약**으로 막고 있었다. 규약은 잊힌다. **파일은 안 잊힌다.**
2. 실행자를 발사하고 `WaitForExit()`으로 기다린다.
3. 종료 시 `outputs/launch/<TaskId>.exit.json`을 남긴다:
   ```json
   { "taskId": "LEDGER-04", "pid": 34288, "exitCode": 0, "exitedAt": "...", "processed": false, "argLength": 2801 }
   ```

**조율자 선(先) 게이트** (프롬프트 §0-A):
- 커밋 레인(`server` `dashboard` `docs` `skills` `outputs/review-log.md`)이 깨끗하고 **`processed: false`인 신호도 없으면 → 즉시 종료. 리포트도 쓰지 않는다.**
- 검수 후 신호의 `processed`를 `true`로 바꾼다.
- `exitCode != 0` → 커밋 금지, HUMAN-INBOX.
- `argLength < 1000` → 프롬프트 잘림 의심(FAIL-2026-013).

## 4. 판단 기준

- **시간이 아니라 사건에 반응한다.** 폴링은 자원을 태우고, 그 자원이 지금 가장 부족하다.
- **`dashboard/data/**`의 런타임 json은 항상 더럽다** — 그것을 "할 일"로 세면 게이트가 영원히 열린다. 레인에서 제외한다.
- **신호가 없다고 실행자가 안 돌았다고 단정하지 마라.** 래퍼 없이 발사된 경우도 있다(과거 회차). 워크트리가 더러우면 평소대로 검수한다. **신호는 추가 정보이지 유일한 진실이 아니다.**

## 5. 부작용과 위험

- **신호 파일이 낡으면 조율자가 오판한다.** 래퍼가 발사 전에 이전 신호를 지운다.
- **래퍼가 죽으면 신호가 안 남는다**(실행자는 살아 있는데). 그때는 워크트리 더러움으로 검수가 트리거되므로 **안전하게 실패한다**(fail-safe).
- 함정 2개를 실제로 밟았다:
  - **PowerShell 5.1은 BOM 없는 UTF-8을 ANSI로 읽는다** → 한글 주석이 깨져 **구문 오류**. BOM을 붙여 해결.
  - **MCP 세션이 자식 프로세스를 함께 죽인다** → `Start-Process`/`Start-Job`으로 띄운 래퍼가 즉사. **`Win32_Process.Create`로 완전 분리 발사**해 해결.

## 6. 되돌림 조건

조율자가 신호를 놓쳐 산출물이 방치되면 → 선(先) 게이트를 끄고 무조건 실행으로 복귀한다(프롬프트 §0-A 삭제). 래퍼는 그대로 둬도 무해하다(프롬프트 파일 전달의 이득은 남는다).
