# 결재 대기 가시성 부족

## 증상

dev-pack 화면에서 결정 게이트의 승인·거절 버튼이 비활성처럼 보였고, 사용자가 이를 기능 오류로 오인했다.

## 원인

- 현재 프로젝트에는 결재 대기 중인 제안이 없었지만, 결정 게이트 패널이 종료된 제안을 기본 표시했다.
- 결재 대기 중인 항목은 ruined-lab 프로젝트에 있었으나, 다른 프로젝트의 대기를 현재 화면에서 확인할 방법이 없었다.
- 사람을 기다리는 상태가 헤더·문서 제목·프로젝트 선택 목록에 드러나지 않았다.

## 해소

- 전 프로젝트를 스캔하는 `/api/inbox`를 추가해 결재·체크포인트·가드레일·악화 대기를 집계한다.
- 헤더에 전역 인박스 배지를 표시하고, 프로젝트 선택 항목에 대기 수를 병기한다.
- 결정 게이트는 대기 중인 제안만 기본 표시하고, 종료된 제안은 접힌 검토 이력으로 이동한다.
- 현재 프로젝트에 대기가 없으면 명시적인 빈 상태를 표시한다.

## 재발 방지

- 결재 대기는 현재 프로젝트에 한정하지 않고 전역 인박스 기준으로 표시한다.
- 종료된 제안에는 승인·거절 버튼을 렌더링하지 않는다.
