# ADR-011 — Phase 0의 완료는 "안전장치가 섰다"가 아니라 **"로컬 AI가 프로그램의 도움으로 DI를 완수할 수 있다"**이다

- 상태: **승인됨 (사람 choi, 2026-07-12)** — 사람이 완료 기준을 직접 재정의했다.
- 관련: `INTENT-DIGEST §4-B`(실행 주체 = 로컬 AI) · `ADR-001`(운영 등급) · `HS-GATE-P00`

## 1. 사람의 정의 (2026-07-12)

> *"로컬과 프로그램으로 그걸 수행할 수 있도록 하네스·스킬·컨텍스트 엔지니어링·프롬프트 엔지니어링을 맞추고, **시뮬레이션까지 돌려서 충분하다 싶어야** 0단계를 마친 것이다."*

**즉 `HS-GATE-P00`은 "검사가 다 있다"로 통과하지 않는다. "로컬 실행자로 한 바퀴가 실제로 돈다"로 통과한다.**

## 2. 상황 — 우리는 층을 혼동하고 있었다 (실측)

| 층 | 실행 주체 | 1회 비용(실측) | 상태 |
| --- | --- | ---: | --- |
| **제품 루프**<br>measure → 제안 생성 → tier1 리뷰 → 결재 → 반입 | **ollama** | 제안 **351 in / 95 out**<br>리뷰 **1,541 in / 147 out** | **이미 로컬이다.** 2026-07-12 LEDGER-04로 되살아났다(`proposal.generated provider=ollama`) |
| **메타 루프**<br>지시서 수행(하네스 제작·코드 수정·검수) | sonnet · 코덱스 | **49,281** (그중 실제 작업 193) | **아직 상위 모델이다** |

**제품은 이미 로컬-퍼스트다. 비계(scaffolding)가 아니다.**
→ **Phase 0의 남은 일은 "메타 루프의 짐을 프로그램이 대신 져서, 로컬 모델도 DI를 수행할 수 있게 만드는 것"이다.**

## 3. 로컬 실행자를 막는 벽 (실측)

1. **ollama는 에이전트가 아니라 생성기다.** `OllamaExecutor.Generate()`가 note 하나를 뱉는다. **도구 호출 루프가 없다.**
2. **컨텍스트 49k.** qwen3:8b는 받지도 못한다. **짐을 줄이는 것이 곧 로컬화다.**
3. **계약 준수 실패.** qwen3:8b가 `functionsWithoutComment`를 `functionsWithOutComment`로 반환했다 — **가장 쉬운 계약에서 실패**했다(ADR-008로 정규화하되 계속 기록).
4. **모델 의존 규약이 남아 있다.** ADR-005 자진신고 · 지시 게이트 되묻기 · verification 문서 작성 · 규칙 준수 거부. **RULES-01에서 sonnet은 금지사항 4개를 거부했다. qwen3:8b가 그럴지는 아직 안 재봤다.**

## 4. 재정의된 `HS-GATE-P00` 통과 조건

**A. 프로그램이 강제한다 (모델 무관)**

- [ ] canonical ID · status enum · 재개 불변식이 **exit code로 강제**된다(State Applier + `handoff-integrity` 확장)
- [ ] WORKSTATE는 **단일 writer**로만 바뀐다
- [ ] **gate manifest**: 명령·순서·**기대 exit**·증거 위치가 기계 파일로 고정(`di-completion-check`)
- [ ] **발사 차단**: Context Pack·claim·transport 검사 실패 시 **child가 시작되지 않는다**
- [ ] **수신 증명**: source hash == replay hash (**모델 무관** — `ADR-010` ✅ 완료)

**B. 짐을 줄인다 (컨텍스트 엔지니어링)**

- [ ] **L0가 20줄 이내**로 Phase·WP·DI·status·blockers·nextActions를 담는다
- [ ] Context Pack의 `readOrder`가 **실제로 턴 수를 줄이는지 측정**한다(비용 = 프리픽스 × 턴)
- [ ] **로컬 모델 컨텍스트 창에 들어가는 지시서 크기**를 실측하고 상한을 정한다(`LAUNCH-BUDGET.json` — **사람 승인**)

**C. 모델 의존 규약을 코드로 옮긴다 (프롬프트 엔지니어링)**

- [ ] 로컬 모델이 **지킬 수 있는 계약**으로 지시서를 다시 쓴다(넓은 판단 → 좁은 선택)
- [ ] 모델 의존 항목 목록(`INTENT-DIGEST §4-B` 오른쪽 열)을 **하나씩 코드/하네스로 대체하거나, 대체 불가하면 그 사실을 명시**한다

**D. ★ 시뮬레이션 (이게 없으면 위 전부가 종이다)**

- [ ] **로컬 실행자로 DI 한 바퀴를 실제로 돌린다.** 게이트 통과율·토큰·턴·실패 사유를 **숫자로** 남긴다.
- [ ] **적대적 시험(RULES-01 계열)을 로컬 모델로 재실행**한다. **금지사항을 거부하는가?**
- [ ] **독립 재개 시험(RESUME-01)을 로컬 모델로 재실행**한다. **L0만 읽고 지어내지 않고 답하는가?**
- [ ] 실패해도 좋다. **실패를 숫자로 남기는 것이 통과 조건이다.** *"충분하다 싶어야"*의 근거가 그 숫자다.

**E. 경계 판정**

- [ ] **v9 `DI-00-01~06` 적합성 행렬**(PASS / PARTIAL / MISSING / NOT_VERIFIED + 증거 경로) — **`DI-00-07`은 "01~06이 끝났다"는 경계 주장이다.** 증명 없이 선언하지 않는다.
- [ ] **사람이 `HS-GATE-P00`을 판정한다.**

## 5. 판단 기준

1. **"검사가 있다"와 "위반 상태로 발사할 수 없다"는 다른 보장이다.** 지금은 전자에 가깝다.
2. **로컬 모델을 세우면 모델 의존 규약이 제일 먼저 무너진다.** 그래서 **시뮬레이션이 필수다** — 안 돌려보면 "될 것 같다"로 끝난다. 이 저장소가 하루 종일 싸운 병이 그것이다.
3. **실패를 숨기지 않는다.** 시뮬레이션이 FAIL하면 **그 숫자가 Phase 0의 산출물이다.** 지표를 green으로 만들려고 기준을 낮추지 않는다(금지사항 1호).

## 6. 되돌림 조건

- 로컬 모델이 **어떤 조건에서도 DI를 완수하지 못한다**는 것이 시뮬레이션으로 증명되면 → **그것 자체가 결론이다.** "로컬 실행자"의 범위를 좁히거나(예: 좁은 판단만 로컬, 코드 작성은 상위 모델) 사람이 목표를 조정한다. **숫자를 보고 사람이 정한다.**
