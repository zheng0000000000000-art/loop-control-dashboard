# FAIL-2026-019 — 빈 인자 `dotnet run`이 **웹서버를 띄워 실행자를 12분간 교착**시켰다

- 상태: 미해결 (`CODEX-GATE-04 §5-5`로 등재)
- 최초 발생일: 2026-07-14
- 최근 발생일: 2026-07-14
- 관련 DI: 06C-1-R1 · GUARD-01 · CODEX-GATE-04
- 구성요소: executor-orchestration
- failureClass: harness_runtime_error
- 심각도: 높음 (**한도 창을 통째로 태울 수 있다.** 사람이 개입하지 않으면 무한 대기)

## 발생 상황

06C-1-R1 실행자(sonnet, PID 16620)가 **39분째인데 12분간 산출물이 없었다.**

**프록시로 단정하지 않고 쟀다:**

```
claude PID 16620 : CPU 20초간 증가 0            ← 일을 안 한다
dotnet PID  7112 : 살아 있음, CPU 증가 0, 11.9분째
  커맨드라인: dotnet run --project server -c Release --no-build --      ← ★ `--` 뒤가 비었다
  자식: LocalFirstWorkflowDashboard.Server.exe (PID 3684)
```

## 관찰된 증상과 영향

**CLI 명령 없이 `dotnet run --project server -- `를 부르면 웹서버가 뜬다. 그리고 영원히 안 끝난다.**

- 실행자는 그 서브프로세스의 **종료를 기다리며 멈춘다.**
- 웹서버는 **stdout을 닫지 않는다.** → **교착.**
- 실행자 프로세스는 **살아 있다.** 죽은 게 아니라서 타임아웃도 안 걸린다.
- **12분 손실.** 검수자가 개입하지 않았으면 **한도 창까지 태웠을 것이다.**

## 발생 이유

- **직접 원인**: 실행자가 인자 없이 `dotnet run ... --`를 호출했다. **왜 그랬는지는 주체 미상**
  (`.out.jsonl`이 종료 시에만 떨어져 실시간 확인 불가. 완료 후 로그에서 확인 필요).
- **★ 근본 원인 — 저장소의 fail-silent 결함**:

  SESSION-BRIEF 함정 #3("미인식 명령이면 조용히 웹서버가 떴다")은 **절반만 고쳐졌다.**
  `GUARD-01`이 **미인식 명령 → exit 2**를 넣었다.
  **그러나 인자가 아예 없는 경우는 "미인식"이 아니다** — **"웹서버 기동"이라는 정상 경로**다.
  **그래서 fail-closed가 발동하지 않는다.**

  > **비대화(headless) 발사 컨텍스트에서 웹서버를 띄우는 것은 언제나 사고다.**

- **기여 원인**: 실행자에게 "빈 인자로 부르지 마라"는 경고가 프롬프트에 없었다.

## 검토한 해결 방법

1. 실행자 프롬프트에 경고만 추가한다 (△ — **프롬프트로 시키지 말고 코드로 강제하라**. CLAUDE.md 철학)
2. **빈 인자를 fail-closed 시킨다** (○)
3. 발사기가 서브프로세스 타임아웃을 건다 (○ — 보완책)
4. 웹서버 기동을 **명시적 의사표시**로만 허용 (○ — 근본)

## 선택한 해결 방법

**2 + 4.** `CODEX-GATE-04 §5-5`에 등재했다.

- **웹서버 기동은 명시적 의사표시를 요구한다** — `serve` 서브커맨드 또는 `--serve` 플래그.
- **인자 없이 부르면 `exit 2` + usage.** 지금은 웹서버가 뜬다.
- **반증 시험**: 비대화 컨텍스트에서 `dotnet run --project server -c Release -- ` (빈 인자) → **exit 2, 프로세스가 끝난다.**

**보완책 (3)**: `run-executor.ps1`이 자식 프로세스에 타임아웃을 걸어야 한다. **미수행.**
