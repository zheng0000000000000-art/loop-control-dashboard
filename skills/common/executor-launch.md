# 스킬: 실행자 발사 — 발사는 "프로세스를 띄웠다"가 아니다

버전: 1 | 도메인: common | 트리거: sonnet 등 헤드리스 실행자를 발사할 때 (조율자·검수자)
| 대상: 지시서를 실행자에게 넘겨 작업시킬 때

## 발사의 정의

**발사 = ①지시 도착 확인 + ②실행 확인 + ③범위 대조.** 셋 중 하나라도 없으면 발사가 아니다.

| 단계 | 무엇을 확인 | 근거 실패 |
| --- | --- | --- |
| ① **지시 도착** | 실행자가 **task ID를 에코백**했는가 | FAIL-2026-013 — 프롬프트가 안 도착해도 실행자는 **저장소를 읽고 알아서 딴 일을 한다** |
| ② **실행** | PID 생존·CPU 누적·산출물 존재 | FAIL-2026-005 — "launched" 문자열은 실행이 아니다 |
| ③ **범위** | 산출물이 지시서 `## 허용 파일 (allowlist)` 안인가 | I-1 — 범위 이탈이 **안전 보류 항목까지** 구현했다 |

## ① 지시 도착 확인 (가장 중요, 가장 늦게 발견됨)

### 프롬프트 전달 — Windows PowerShell

```powershell
# ❌ 틀림 — 배열 인자는 공백에서 쪼개진다. -p 가 첫 토큰만 받는다.
Start-Process claude.exe -ArgumentList @("-p", $prompt, "--dangerously-skip-permissions")

# ✅ 맞음 — 명시적으로 인용된 단일 인자 문자열
$argline = '-p "' + $prompt.Replace('"','\"') + '" --dangerously-skip-permissions'
Start-Process claude.exe -ArgumentList $argline -RedirectStandardOutput $log ...
```

- **프롬프트는 한 줄로** 쓴다(줄바꿈은 인자 경계를 깨뜨린다). 상세는 **지시서 파일**에 두고 프롬프트는 그 경로만 가리킨다.
- 전달 방식을 바꿨으면 **반드시 프로브로 재검증**한다(아래).

### ACK 에코백 — 필수

프롬프트 첫 지시를 이렇게 한다:

> `task ID <DI>. 맨 첫 줄에 정확히 ACK-<DI> 라고 출력하라. 그 다음 <지시서 경로> 를 읽고 수행하라.`

**실행자 출력에 `ACK-<DI>`가 없으면 발사 실패다. 산출물을 폐기하고 재발사한다.**
프롬프트가 잘리면 ACK도 안 나오므로 **잘림이 자동 검출된다.**

### 프로브 (전달 방식 검증용, 부작용 0)

```
task ID PROBE-<난수>. 맨 첫 줄에 정확히 ACK-PROBE-<난수> 라고만 출력하라. 파일은 절대 수정하지 마라.
```
ACK가 돌아오면 전달 OK. 안 돌아오면 **아무것도 발사하지 마라.**

## ② 실행 확인
PID 파일 기록 + 프로세스 생존·CPU 확인. **`claude.exe` StartTime으로 판정하지 마라**(Cowork 앱과 혼동 — 데드락 유발).

## ③ 범위 대조
완료 후 `git status`를 지시서 allowlist와 대조한다(`scope-check` 하네스, HS-06).
**범위 밖 수정이 있으면 산출물 전체 반려 + 격리 후 되돌림.** 되돌리기 전 `outputs/quarantine/`에 보존한다.

## 발사 전 게이트 (조율자)
`gate-clean server` exit 0 / 실행 중 sonnet 없음 / 진행 항목 커밋 존재 / 다음 대기 항목 존재.
**raw `git status`로 clean을 판정하지 마라** — 줄바꿈 표현차로 게이트가 영구 잠긴다(FAIL-2026-010).

## 하지 않는 것
발사 자체는 **사람 배치 게이트**다(현행). 결재·반입·push는 실행자도 조율자도 대행하지 않는다.

## 한 줄
**프로세스를 띄운 것은 발사가 아니다. 지시가 도착한 것이 발사다.**
