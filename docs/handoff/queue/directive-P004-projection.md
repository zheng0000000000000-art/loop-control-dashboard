# P0-04 — Projection 생성기 + WORKSTATE 해시 스탬핑 (`dotnet run -- projection`)

이 지시서는 `docs/directives/_header.md`의 불변 제약과 **공통 검수 기준**을 따른다. 작업 시작 전 먼저 읽어라.
유형: implementation. 근거: 계획서 §0.5 — **"`HANDOFF.md`와 `RUNTIME-INDEX.md`는 WORKSTATE에서 생성되는 Projection이며 직접 상태 원본으로 편집하지 않는다."** 그리고 **`WORKSTATE.changedFiles`는 `{"path","sha256"}` 형식이어야 한다.**

## 왜 하는가 (실측 근거 — 추측 아님)

1. **손으로 쓴 상태 문서는 낡는다.** `docs/STATUS.md`가 "ORCH-01 참조 .cs 준비됨"이라고 적고 있었으나, 그 파일은 커밋 `797e7bc`가 이미 삭제한 상태였다. 발사 직전에 사람이 손으로 발견했다 — **하네스 6개가 돌고 있었는데 아무도 못 잡았다.**
2. **인수인계가 기계 검증 불가능하다.** 코덱스가 만든 `handoff-integrity` 하네스가 첫 실행에서 잡았다:
   ```
   handoff-integrity → exit 1
   "changedFiles item lacks sha256/hash" × 3
   ```
   `WORKSTATE.changedFiles`에 경로만 있고 해시가 없다. 다음 세션은 "그때 그 파일이 지금 이 파일인가"를 확인할 방법이 없다.
3. **해시를 LLM에게 적으라고 시키면 안 된다.** 프로젝트 철학: *"LLM은 적게 기억하고 적게 생성한다. 프로그램이 많이 기억하고 조립하고 검증한다."* **프로그램이 계산해서 채운다.**

## 목표

읽기+쓰기 CLI `projection`을 만든다. 두 가지를 한다.

**(1) WORKSTATE 해시 스탬핑**
- `docs/handoff/WORKSTATE.json`의 `changedFiles` 각 항목에 대해 **파일을 읽어 sha256을 계산**해 채운다.
- 형식: `{"path": "server/Foo.cs", "sha256": "abc123...", "linesBefore": N, "linesAfter": M}` — 기존 필드는 보존하고 `sha256`만 추가/갱신한다.
- **파일이 존재하지 않으면** `sha256`을 `null`로 두고 `missing: true`를 표시한다. **삭제를 숨기지 마라.**
- 원자적 쓰기: `WORKSTATE.tmp` 작성 후 replace(계획서 §0.5).

**(2) Projection 생성**
- `docs/context/RUNTIME-INDEX.md` (**L0** — 계획서 §0.6): 현재 `phaseId`/`wpId`/`diId`/`status`, blocker, 다음 작업. **이것만 읽어도 현재 위치를 안다.** 20줄 이내로 짧게.
- `docs/handoff/HANDOFF.md` (사람용 Projection): 현재 위치 · 완료 · 미완료 · 불변식 · 변경 파일과 hash · 알려진 실패 · 다음 작업 순서 · 재개 전 검증.
- **두 파일 머리에 반드시 넣는다**: `<!-- GENERATED by 'dotnet run --project server -- projection'. 직접 편집하지 마라 — 다음 실행에서 덮어써진다. 원본은 docs/handoff/WORKSTATE.json 이다. -->`

**(3) 안전 불변**
- **WORKSTATE의 의미를 바꾸지 마라.** 해시만 채운다. `status`·`diId`·`nextActions`를 임의로 고치지 않는다.
- git commit/push 금지. 결재·발사 금지.
- 자기 hash 자기 참조 금지: Projection 파일 자신의 hash를 WORKSTATE에 기록하지 않는다(계획서 §0.5 순환 참조 금지).

## 작업
1. `server/ProjectionCli.cs` 신규. `server/Cli/CliRouter.cs TryRun`에 `projection` 분기 등록(기존 CLI 분기 패턴 그대로).
   - **`server/Harness/**` 는 코덱스 배타 영역이다. 절대 건드리지 마라** — 지금 미커밋 변경이 있어도 그대로 두라. `projection`은 하네스가 아니라 생성기다.
2. `docs/context/` 디렉터리를 만들고 `RUNTIME-INDEX.md`를 생성한다.
3. `docs/handoff/HANDOFF.md`를 생성한다.
4. 함수마다 **한국어 기능 주석 1줄**(`functionsWithoutComment`가 올라가면 실패다).

## 검수 기준 (공통 항목 + 아래)
1. `dotnet run --project server -c Release -- projection` → exit 0. `WORKSTATE.changedFiles` 전 항목에 `sha256`이 채워진다(없는 파일은 `missing: true`).
2. **`dotnet run --project server -c Release -- handoff-integrity` → exit 0**. 지금은 `missing-hash`로 exit 1이다. **이 하네스를 통과시키는 것이 이 작업의 존재 이유다.**
3. `RUNTIME-INDEX.md`와 `HANDOFF.md`가 생성되고, 머리에 GENERATED 경고가 있다.
4. **재실행 안전(idempotent)**: 두 번 연속 실행해도 결과가 동일하다(diff 없음).
5. `measure dev-pack` 비악화(현재 위반 1 — 코덱스 몫이며 네 책임 아니다. **네가 새 위반을 만들지 마라**).
6. `verify-behavior` → `behaviorEqual: true`.
