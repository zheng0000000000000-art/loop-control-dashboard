# ADR-007 — 주체별 세션 수명 정책: 코덱스는 매 회차 이사, 검수자는 문서로 이사

- 상태: 승인됨 (사람: choi, 2026-07-11)
- 일시: 2026-07-11
- 제안: 검수자 세션(Claude)
- 근거 문서: 계획서 §0.6(컨텍스트 절약) · §0.5(저장소가 유일한 인수인계 원본)

## 1. 상황 (실측)

| 주체 | 컨텍스트 누적 | 근거 |
| --- | --- | --- |
| sonnet 실행자 | ❌ 없음 | 발사마다 새 프로세스(1 DI = 1 세션) |
| 조율자 | ❌ 없음 | 스케줄 실행마다 새 세션. 매 회차 저장소를 새로 읽는다 |
| **코덱스** | ⚠️ **누적** | `automation.toml`에 `target_thread_id = "019f4c36-..."` 고정. 15분마다 **같은 스레드**에 append. 하루 49회차 |
| **검수자** | ⚠️ 누적 | 대화 세션. 하루 종일 자란다 |

프롬프트에 "이전 대화 무시하라"고 써도 **스레드 히스토리는 매 회차 토큰을 먹는다.** 그리고 코덱스는 이미 `SESSION-*.md`로 회차 인계를 하고 있다 — **스레드를 유지할 이유가 없었다.**

## 2. 선택지

(A) 그대로 둔다 — 컨텍스트가 계속 자라고, 낡은 맥락이 판단을 오염시킨다.
(B) **주체별 수명 정책을 세운다**: 누적되는 둘만 이사시킨다.
(C) 전체를 매번 초기화 — 조율자·sonnet은 이미 그렇다. 추가 작업 불필요.

## 3. 선택 — (B)

| 주체 | 정책 |
| --- | --- |
| sonnet | 1 DI = 1 세션 (현행 유지) |
| 조율자 | 1 회차 = 1 세션 (현행 유지) |
| **코덱스** | **매 회차 새 스레드.** `target_thread_id` 제거. 인계는 `docs/handoff/sessions/` 최신 SESSION 문서 **하나**로만 |
| **검수자** | **`docs/handoff/REVIEWER-HANDOFF.md`로 이사.** 판단할 때마다 갱신한다 — 세션 종료 시점이 아니다. **세션은 예고 없이 죽는다.** |

## 4. 판단 기준

- **저장소가 유일한 인수인계 원본이다**(계획서 §0.5). 스레드 히스토리에 의존하는 순간 그 원칙이 깨진다.
- **이사가 안전하려면 "저장소만 읽고 재구성 가능"해야 한다.** 그래서 P0-03(handoff-integrity)·P0-04(Projection/L0)·P0-05(Context Pack)가 **선행 조건**이다. 그것 없이 이사시키면 이사가 아니라 **기억상실**이다.
- 이사 **시점**을 기계로 판정하려면 토큰을 재야 한다(ADR-006). 지금은 못 잰다 — 그래서 코덱스는 **무조건 매 회차**, 검수자는 **판단마다 갱신**이라는 단순 규칙을 쓴다. **느낌으로 "슬슬 길어졌다" 판단하지 않는다.**

## 5. 결과

- `~/.codex/automations/codex-15-qa/automation.toml`: `target_thread_id` 제거(백업 `.bak-20260711`), 프롬프트 첫머리에 이사 규약 삽입. **TOML 파서로 검증 완료**(작업 중 이스케이프 안 된 따옴표로 파일을 깨뜨릴 뻔했다 — 실체 검증이 잡았다).
- `docs/handoff/REVIEWER-HANDOFF.md` 신설(검수자 전용, 상시 최신).
- 검증: 다음 코덱스 회차(:37 또는 :52)에 SESSION 문서가 정상 생성되고 큐를 픽업하면 이사 성공. **실패하면 백업 toml로 즉시 복구.**

## 6. 되돌림 조건

코덱스가 새 스레드에서 맥락을 못 이어 회차 품질이 떨어지면(SESSION 문서만으로 부족), `target_thread_id`를 복구하고 대신 **N회차마다 이사**로 완화한다.

## 7. 관련 실패 사례

- FAIL-2026-013 (지시 미도착 — 실행자는 지시서를 받은 적이 없는데 저장소를 읽고 알아서 딴 일을 했다). **이사의 위험을 보여준다: 인계 문서가 부실하면 주체는 추측으로 채운다.**
