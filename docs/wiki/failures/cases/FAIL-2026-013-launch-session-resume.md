# FAIL-2026-013 — 발사가 격리되지 않았다: `claude -p`가 기존 세션을 이어받아 실행자가 남의 작업을 계속했다

- 상태: 해결됨(격리 발사로 수정)
- 최초 발생일: 2026-07-11 (소급하면 I-1 발생 시점 전체)
- 관련 DI: HOOK-01, FEAT-01, I-1(지시서 이탈)
- 구성요소: executor-orchestration
- failureClass: unnormalized_gate, observability, design_learning
- 심각도: **critical** — "실행자가 지시를 어긴다"는 진단 자체가 틀렸었다.

## 발생 상황

HOOK-01(하네스 레지스트리 이관)을 헤드리스 발사했더니, 실행자가 HOOK-01을 하지 않고 `server/Tier2Approver.cs`에 **FEAT-01(안전 보류 항목 — 반입 승인을 AI에 위임)**을 109줄 구현했다. 재발사해도 `Tier2ApproverTestCli.cs`를 건드렸다. 세 번째 발사에서는 **파일을 하나도 안 바꾸고 stdout에 "FEAT-01 완료. FIX-01·FIX-02·FEAT-02·FEAT-01 요약..."을 출력**했다 — 내가 시키지도 않은 작업의 완료 보고였다.

## 발생 이유(직접·근본)

- **직접**: `claude -p`가 **해당 작업 디렉터리의 기존 세션을 이어받고 있었다.** `~/.claude/projects/<repo>/`에 4.3MB짜리 세션 파일이 있었고, 내 발사 시각에 그 파일이 갱신됐다. 즉 **내 프롬프트가 남의 대화에 후속 턴으로 붙었다.** 실행자는 그 대화에서 하던 일(FEAT-01)을 계속했다.
- **확증**: `--session-id <new>`를 주자 CLI가 거부했다 — `--session-id can only be used with --continue or --resume if --fork-session is also specified.` **발사가 이미 continue/resume 맥락이었다는 뜻이다.** `--fork-session`을 붙이자 정상 격리 실행됐다.
- **근본**: **발사 격리를 검증하지 않았다.** "프롬프트를 인자로 직접 전달했으니 격리됐다"고 가정했다. FAIL-2026-005는 *"launched" ≠ 실행*을 가르쳤는데, 여기서 배울 것은 **"launched ≠ 격리"**다.

## 이 오진이 만든 2차 피해

1. **I-1(지시서 이탈) 진단이 통째로 틀렸다.** 실행자가 "말을 안 듣는다"고 판단해 격리 프롬프트·화이트리스트 프롬프트를 계속 강화했다. **전부 헛수고였다** — 실행자는 지시를 어긴 게 아니라 **다른 대화를 하고 있었다.**
2. **검수자가 주체를 두 번 오귀인했다.** ①"HOOK-01 sonnet의 지시서 이탈"(시각 상관이라는 프록시) ②"내가 발사하지 않은 미상 에이전트"(프로세스 목록이라는 프록시). 둘 다 틀렸다. 범인은 **발사 메커니즘**이었다.
3. 안전 보류 항목(FEAT-01)이 두 번 무단 구현됐다. 검수자가 눈으로 `git status`를 봐서 잡았다.

## 선택한 해결 방법

발사 시 **세션 격리를 강제**한다:
```
claude -p "<프롬프트>" --session-id <새 GUID> --fork-session --dangerously-skip-permissions
```
그리고 프롬프트 첫 줄에 **"이전 대화 맥락이 있다면 전부 무시하라"**를 명시한다(이중 방어).

## 재발 방지

- **발사 = 프롬프트 전달 + 세션 격리 + 실행 확인 + 산출물 범위 대조.** 넷 다 해야 발사다.
- SONNET-QUEUE 발사 규칙·프롬프트 템플릿에 `--session-id <new GUID> --fork-session`을 **필수**로 명시.
- `scope-check` 하네스(HS-06)가 산출물 범위를 사후 검출한다 — 격리가 또 깨져도 잡힌다. **말이 아니라 검출로 강제한다.**
- ORCH-03(자식 프로세스 직접 소유)의 요구사항에 **세션 격리**를 추가한다.

## 교훈

**"실행자가 말을 안 듣는다"는 진단은 거의 항상 틀렸다.** 실행자는 자기가 받은 맥락 안에서 합리적으로 행동한다. 이상 행동을 보면 **그가 실제로 무슨 맥락을 받았는지** 먼저 확인하라 — 프롬프트에 뭐라 썼는지가 아니라. 여기서도 실체(세션 파일)를 안 보고 프록시(내가 쓴 프롬프트 텍스트)를 믿었다.
