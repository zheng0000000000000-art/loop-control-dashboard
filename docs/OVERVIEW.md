## 1. 한 문장

**AI가 코드를 고치고, 다른 AI가 그걸 검수하고, 프로그램이 그 전부를 강제하는 루프**를 만든다.
사람은 **결재선**(발사·push·기준 변경·이양)에만 남는다.

### 그 밑에 깔린 철학 (판단이 갈리면 이 문장이 심판이다)

> **LLM은 적게 기억하고 적게 생성한다. 프로그램이 많이 기억하고 조립하고 검증한다. LLM은 판단만 한다.**

**"프롬프트로 시킨다" vs "코드로 강제한다"에서 항상 후자.**
실증: 발사 프롬프트의 `ACK` 지시가 **5회 중 5회 무시**됐다. **말은 안 지켜지고, 검사만 지켜진다.**

## 3. 구조 — 지금 실제로 돌아가는 것

### 3.1 주체 다섯 (역할이 갈려 있다)

| 주체 | 무엇을 하나 | 무엇을 못 하나 |
| --- | --- | --- |
| **사람(choi)** | 발사 승인 · push · 기준 변경 결재 · HS-GATE 판정 | — |
| **검수자**(대화 세션) | 지시서 작성 · 발사 · **하네스 직접 재실행해 판정** · ADR · 기록 | 하네스 제작 금지(만든 사람이 검증하면 같은 착각을 코드에 새긴다) |
| **조율자**(5분 트리거, 사건 기반) | 산출물 검수 · **로컬 커밋만** · HUMAN-INBOX 등재 | **발사 금지 · push 금지 · 결재 대행 금지** |
| **코덱스**(15분) | **하네스·스킬 제작** · FAIL 위키 · QA | `server/Harness/` 밖 쓰기 금지 |
| **실행자(sonnet)** | 지시서 1건 = 프로세스 1개. 코드 수정 | allowlist 밖 수정 금지 · commit/push/결재 금지 |

**핵심 분리(ADR-002)**: **제작 = 코덱스 / 검사 = 조율자 / 판정 = 검수자 / 결재 = 사람.**
**자기보고는 증거가 아니다.** 검수자는 실행자가 적은 하네스를 **직접 다시 돌려 대조한다.**

### 3.2 하네스 — 규칙을 강제하는 프로그램 (판정은 exit code)

```
build-verify · measure dev-pack · verify-behavior      → 코드 품질·동작 보존
gate-clean                                             → 트리 clean 판정(줄바꿈 표현차로 잠기지 않게)
scope-check     (+ FILE-CLAIMS 확장)                   → allowlist 밖 수정 + 실행 중 파일 충돌
handoff-integrity                                      → WORKSTATE가 실체와 일치하는가(sha256)
context-pack-integrity                                 → 지시서의 참조 입력이 사라졌거나 낡았는가
doc-integrity · claim-check · hs-scan                  → 문서 잘림 · 허위 완료 주장 · 승격 심사 트리거
projection                                             → WORKSTATE → L0/HANDOFF 생성(손편집 금지)
```

정본 목록: `docs/handoff/HARNESSES.md`. **출력 문자열·정규식으로 성패를 세지 마라. exit code로 판정한다.**

### 3.3 데이터 — 기계가 읽는 상태

| 파일 | 무엇 | 쓰는 주체 |
| --- | --- | --- |
| `docs/handoff/WORKSTATE.json` | **상태 원본**(phase/DI/status/changedFiles+sha256) | 실행자 → `projection`이 스탬핑 |
| `docs/context/RUNTIME-INDEX.md` | **L0** — 기계가 생성한 현재 위치 | `projection`만 |
| `docs/handoff/FILE-CLAIMS.json` | 누가 지금 어느 파일을 잡고 있나 | **발사 래퍼만**(하네스는 읽기만) |
| `outputs/launch/usage-ledger.jsonl` | **상위 모델 토큰·비용 원장** | 발사 래퍼(append만) |
| `dashboard/data/*/run-log.json` | 런타임 이벤트 + **ollama 토큰** | 서버 |
| `dashboard/data/*/blueprint.json`·`workflow-definition.json` | **기준 파일 — 사람 결재** | 사람만 |

### 3.4 발사 경로 (`outputs/launch/run-executor.ps1`)

```
지시서 allowlist 파싱
  → claude -p --output-format json 발사        ← ⚠ 여기서 프로세스가 먼저 뜬다
  → FILE-CLAIMS에 claim 등록(active)           ← ⚠ 그 '뒤에' 예약한다 (무주공산 구간)
  → 종료 대기
  → claim 해제(released) + usage(토큰·비용) 기록 + 완료 신호(sentinel)
```

**조율자는 그 신호가 있을 때만 깨어난다**(ADR-009). 5분마다 헛도는 세션을 없앴다.

> ### ⚠ 이 경로의 알려진 결함 3개 (2026-07-12 외부 검수 + 검수자 실측)
>
> 1. **프롬프트는 여전히 명령행 인자로 간다.** 파일은 **원본일 뿐**이고, 래퍼가 줄바꿈을 지워 `-p "..."`로 재조립한다.
>    → 인자 길이·quote 경계·구조(코드펜스/JSON) 손실 위험이 **남아 있다.** **"FAIL-2026-013을 구조적으로 제거했다"는 과장이었다.**
>    → 필요한 것: **stdin 전달** + **source hash ↔ 수신 receipt hash 일치 증명**.
> 2. **claim이 사전 예약이 아니라 사후 관측이다.** 프로세스를 띄운 **뒤에** 등록하므로 그 사이 충돌을 못 잡는다.
>    → 필요한 것: **발사 전 원자적 예약**, 충돌이면 **child를 아예 띄우지 않고 exit 1**.
> 3. **발사 전 strict preflight가 없다.** `context-pack-integrity`는 존재하지만 **위반 상태에서도 발사가 된다.**
>    → **"검사할 수 있다"와 "위반 상태로 발사할 수 없다"는 다른 보장이다.**

## 7. 이 저장소에서 일하는 방식 (일곱 줄)

1. **판정은 exit code로.** 출력 문자열·정규식으로 성패를 세지 마라.
2. **프록시로 원인을 단정하지 마라.** 커밋 접두사·타임스탬프 상관·에러 문구는 증거가 아니다. **모르면 "주체 미상"이라고 쓴다.**
3. **통과를 믿기 전에 실패시킬 수 있음을 먼저 증명해라.** 하네스를 만들었으면 **장애 주입**부터 한다.
4. **볼 데이터가 없으면 하네스를 만들지 마라.** 데이터를 만드는 것이 선행 과제다(`gate-audit`이 이걸 안 물어서 철회됐다).
5. **자기보고는 증거가 아니다.** 검수자가 직접 재실행해 대조한다.
6. **지표를 목표로 삼지 마라.** verification 문서에 **`## 지표는 만족했으나 목적은 미달인 부분`**을 자진 신고한다. **신고하면 감점이 아니다. 숨기면 반려다.**
7. **기록 파일은 append만.** 통째로 다시 쓰면 이미 깨진 글자가 조용히 오염된다.

## 8. 알려진 부채 (숨기지 않는다)

- **`handoff-integrity`는 스탬핑 주체(`projection`)와 검증 주체가 같다.** 재실행하면 **언제든 green을 제조할 수 있다.** 드리프트 탐지 목적에서는 유효하나, **HS-GATE 판정 시 사람이 알고 있어야 한다.**
  (`context-pack-integrity`·`scope-check`에는 이 결함을 넣지 않았다 — **쓰는 자와 검사하는 자를 분리했다.**)
- **저장소에 이미 깨진 한글이 박혀 있다**(`WORKSTATE.json`의 note 등). **U+FFFD가 아니라 이중 디코딩된 정상 유니코드라 자동 검출이 안 된다.**
- **`CLAUDE.md`를 압축한 뒤 규칙이 여전히 지켜지는지 검증하지 않았다.** 줄이다가 안전장치를 깎았으면 절감이 아니라 손실이다.
- **`hs-scan` exit 1**(승격 후보 14건) — 심사되지 않은 채 쌓여 있다.
- **이름 체계가 넷이다**(`ROOT n` / `Axis n` / `P0~P4` / `Phase·WP·DI`). 매핑표는 Phase 0 이후.
