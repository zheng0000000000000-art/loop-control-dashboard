## 배경 (읽고 시작해라)

1. `docs/handoff/decisions/ADR-010-transport-receipt.md` — 이 작업의 결정 근거
2. `server/Harness/LaunchCheckCli.cs` — **코덱스가 이미 새 판정 규칙을 만들어뒀다.** 지금은 evidence 파일이 없어서 **exit 1**이다. **네 작업이 그걸 exit 0으로 뒤집는다.**
3. `outputs/launch/run-executor.ps1` — **네가 고칠 것**

**현재 문제**: 발사 래퍼가 프롬프트를 **명령행 인자**(`-p "..."`)로 조립해 넘긴다. 인자 길이·quote 경계·구조 손실 위험이 남아 있고, **실행자가 실제로 무엇을 받았는지 증명할 방법이 없다.**

**검수자가 실측으로 확인했다**: 설치된 Claude CLI가 `--replay-user-messages`를 지원한다. stdin으로 보낸 user 메시지를 **CLI가 그대로 되돌려준다.** 그 sha256을 대조하면 **모델의 협조 없이 전송 무결성이 증명된다.**

## 목표

`run-executor.ps1`을 **stdin 전달 + replay 수집 + transport evidence 생산**으로 바꾼다.

### 1. 프롬프트를 명령행에서 완전히 제거한다

```powershell
claude.exe -p --verbose --input-format stream-json --output-format stream-json --replay-user-messages --dangerously-skip-permissions
```

- 프롬프트 본문은 **stdin**으로만 보낸다. `-p` 뒤에 본문을 붙이지 마라.
- stdin payload는 **stream-json 한 줄**: `{"type":"user","message":{"role":"user","content":"<본문>"}}`

### 2. ★ UTF-8 바이트를 직접 써라 (여기서 이미 한 번 당했다)

**PowerShell 5.1에는 `ProcessStartInfo.StandardInputEncoding`이 없다.** 기본 writer를 쓰면 **한글이 깨진다.** 실제로 첫 시도에서 `ACK�� ����ض�`로 도착했다.

```powershell
$bytes = [Text.Encoding]::UTF8.GetBytes($payloadLine + "`n")
$proc.StandardInput.BaseStream.Write($bytes, 0, $bytes.Length)
$proc.StandardInput.BaseStream.Flush()
$proc.StandardInput.Close()
```

출력도 `$psi.StandardOutputEncoding = New-Object System.Text.UTF8Encoding($false)` 로 받아라.

### 3. replay 이벤트를 **구조적으로** 파싱한다

터미널 문자열 검색 금지. **JSONL 각 줄을 파싱해 `type == "user"` 이벤트를 찾는다.**
`message.content`가 문자열이면 그대로, 블록 배열이면 `text`를 이어붙여 본문을 복원한다.

### 4. transport evidence를 남긴다 (계약은 코덱스가 이미 고정했다)

`outputs/launch/<TaskId>.transport.json`:

```json
{
  "schemaVersion": 1,
  "taskId": "...",
  "cliVersion": "...",
  "sourceSha256": "<64자리 전체>",
  "payloadSha256": "<64자리 전체>",
  "replaySha256":  "<64자리 전체>",
  "sourceByteLength": 0,
  "payloadByteLength": 0,
  "replayByteLength": 0,
  "replayEventCount": 1,
  "pid": 0,
  "startedAt": "...",
  "exitedAt": "...",
  "verdict": "TRANSPORT_VALID | TRANSPORT_INVALID"
}
```

- **sha256은 64자리 전체.** 축약 금지.
- **`sourceSha256`** = 프롬프트 **파일** 원본의 해시. **`payloadSha256`** = stdin으로 실제 보낸 **본문**의 해시. **둘은 다를 수 있다**(줄바꿈 처리 등) — **그래서 둘 다 남긴다.**
- **프롬프트 본문 자체를 evidence나 usage ledger에 복제하지 마라.** 해시와 길이만.

### 5. fail-closed

- replay 이벤트 **없음 / 2개 이상 / 파싱 실패 / hash 불일치 / CLI 옵션 미지원** → `verdict: TRANSPORT_INVALID`
- **그 경우 실행자 산출물을 유효한 것으로 취급하지 마라.** sentinel에 `transportValid: false`를 남기고 **조율자가 커밋하지 않도록** 한다.
- **명령행 방식으로 조용히 폴백하는 경로를 남기지 마라.** 실패하면 실패로 끝낸다.

## 하지 않는 것 (범위 밖)

- ❌ FILE-CLAIMS 예약 순서 변경(claim이 프로세스 시작 뒤에 등록되는 문제) — **별도 DI다.**
- ❌ WORKSTATE·상태 전이 관련 변경 — **별도 DI다.**
- ❌ `docs/handoff/LAUNCH-BUDGET.json` 정책값 창작 — **사람 승인 사항.** 없으면 없는 대로 둔다.

## 실체 증명 (stdout에 원문으로)

1. **한글·emoji·따옴표·역슬래시·여러 줄**이 섞인 프롬프트로 실제 발사 → `payloadSha256 == replaySha256` **일치**를 보여라.
2. `dotnet run --project server -c Release -- launch-check <TaskId> outputs/launch/<TaskId>.transport.json` → **exit 0** (지금은 evidence가 없어 exit 1이다. **네 작업 전후로 1 → 0이 되는 것이 증명이다.**)
3. **명령행에 프롬프트 본문이 없음**을 증명해라: 실행 중 프로세스의 CommandLine을 떠서 보여라(`Get-CimInstance Win32_Process`). 본문이 보이면 실패다.
4. **깨진 인코딩 fixture**: PowerShell 기본 writer로 보내면 **mismatch로 잡히는가**를 보여라(사본/임시 경로에서. 실제 발사 아님).

**값을 지어내지 마라.** 증명 못 하면 `## 지표는 만족했으나 목적은 미달인 부분`에 그대로 써라.
